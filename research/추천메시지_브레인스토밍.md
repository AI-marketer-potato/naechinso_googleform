# 추천 메시지 브레인스토밍
> 핵심 관점: "추천사"가 아니라 "친구한테 보내는 메시지"
> 마지막 업데이트: 2026-03-25

---

## 전제

지금까지 "추천사"라고 불렀지만, 이 단어 자체가 문제다.
추천사 → 공식적이고, 잘 써야 한다는 압박이 생김

바꿔야 할 프레이밍:
```
추천사 (formal)   →   친구한테 보내는 메시지 (casual)
리뷰를 작성한다   →   카카오로 친구한테 연락하는 것
```

핵심 질문: "이 사람 어때?" 라고 친구한테 카카오로 보내는 것과
           내친소 안에서 소개하는 것이 같은 느낌이어야 한다

---

## 1. 메시지 형태 — 어떤 방식으로 전달할 수 있나

### 1-1. 이모지/반응 하나

- 설명 없이 이모지 하나만 보내는 것
- 예시: 💘 / 👀 / 🔥 / 😂
- 받는 사람 입장: "얘가 나한테 이 사람 보내면서 불꽃 이모지 달았어" — 충분히 맥락이 전달됨
- 레퍼런스: iMessage Tapback (1탭으로 반응), BeReal RealMoji (반응 사진 1장)
- PO 포인트: Facebook Reactions 기준 이모지가 있으면 공유율 33% 상승, 상호작용 57% 증가
- 적합한 맥락: 세렌디피티형 ("보다가 생각났어" — 설명이 필요 없는 순간)

### 1-2. 태그 선택

- 미리 준비된 태그를 1~3개 탭해서 보내는 것
- 예시 태그: `#네스타일` `#유머있음` `#진지한만남원해` `#취향저격`
- 받는 사람 입장: 친구가 어떤 포인트를 봤는지 바로 이해됨
- 레퍼런스: KakaoTalk 스티커 (감정 표현이 아니라 "어떻게 보이고 싶은가"의 표현), Letterboxd 별점만 남기기
- PO 포인트: 태그 선택은 1초 이내 결정 가능 — 텍스트 입력 대비 이탈률 대폭 낮음
- 적합한 맥락: 둘 다 (능동형/세렌디피티형 모두)

### 1-3. 짧은 한 줄 텍스트

- 자유 입력이지만 글자 수 제한 (예: 최대 30자)
- 예시: "진짜 네 스타일인데ㅋㅋ" / "얘 생각났어" / "진지한 것 같아서"
- 받는 사람 입장: 친구 목소리로 들림 — 알고리즘 추천과 완전히 다른 감정
- 레퍼런스: TikTok "send to friend" (메시지 입력 선택 사항, 빈 칸으로 보내도 됨), Instagram DM 공유
- PO 포인트: 글자 수 제한이 역설적으로 부담을 낮춤 — "길게 안 써도 된다"는 신호
- 적합한 맥락: 능동형 (친구를 위해 일부러 찾은 경우 — 한마디 더 하고 싶은 마음)

### 1-4. 프리셋 문장 선택 (편집 가능)

- 미리 작성된 문장을 선택하면 내 이름으로 전송됨
- 예시 프리셋:
  - "보다가 네 생각났어"
  - "이 사람 성격 좋을 것 같아"
  - "취향 저격 아니야?"
  - "진지하게 만나볼 만한 것 같아서"
- 받는 사람 입장: 프리셋인 걸 알아도 친구가 골랐다는 사실이 의미 있음
- 레퍼런스: AngelList 소개 템플릿 자동 작성 (응답률 ~50%), Spotify 공유 메시지
- PO 포인트: AngelList 연구 — 소개자(middleman)의 마찰을 줄이는 것이 전환율의 가장 큰 레버
- 적합한 맥락: 능동형 + 빠른 세렌디피티형

### 1-5. 음성 메시지

- 15초 이내 음성 한 마디 녹음 후 전송
- 예시: "야 이 사람 봐봐, 진짜 네 스타일인 것 같아서~"
- 받는 사람 입장: 친구 목소리가 들림 — 신뢰도와 온기가 가장 높음
- 레퍼런스: WhatsApp 음성 메시지 (문자보다 강한 사회적 유대 형성), KakaoTalk 음성 메시지
- PO 포인트: 음성은 사회적 유대를 가장 강하게 만드는 포맷 (연구 consensus). 단, 68%가 2번 들어야 내용 파악 → 첫 소개보다 후속 대화에 더 적합
- 적합한 맥락: 능동형 (정성 들인 소개, 오랜 친구 사이)

## 2. 메시지를 보내는 타이밍 — 언제 자연스럽게 발생하나

### 2-1. 피드 스크롤 중 즉시

- 카드를 보다가 → 공유 버튼 → 친구 선택 → 이모지 or 태그 → 3초 안에 완료
- 레퍼런스: TikTok/Instagram DM 공유 (현재 화면을 벗어나지 않고 전송)
- PO 포인트: 화면 이탈이 생기면 공유 행동 80% 이상 소실
- 설계 원칙: 공유 버튼 탭 → 친구 선택 → 이모지 선택 → 전송까지 화면 전환 0회

### 2-2. 친구 프로필 방문 중

- 앱 안에서 친구 프로필을 보다가 → "OO한테 어울릴 것 같은 사람 찾아줄까?" → 능동형 탐색 시작
- 레퍼런스: LinkedIn "2촌 프로필 → 소개 요청" 진입점
- PO 포인트: 친구를 먼저 생각하는 맥락에서 진입 → 메시지 퀄리티가 올라감

### 2-3. 매칭 성사 후 친구에게 알림

- 내가 소개해준 두 사람이 연결됨 → "OO와 XX가 연결됐어! 네 덕분이야" 알림
- 알림에서 바로 "잘됐다 🎉" 한 탭 반응 가능
- 레퍼런스: Wingman 매칭 성사 알림, Hinge "We Met" 후속 질문
- PO 포인트: 성사 순간이 가장 강력한 재공유 트리거 — 이 순간에 "친구 더 소개해주기" 유도

### 2-4. 앱 밖에서 시작 (카카오 딥링크)

- 카카오에서 "이 사람 어때?" 를 받은 비유저 → 링크 클릭 → 설치 → 프로필 확인
- 레퍼런스: Locket (앱 없으면 사진 못 봄 → 설치 유도), Hinge Matchmaker 초대 링크
- PO 포인트: "이미 뭔가가 기다리고 있어" 프레이밍이 단순 초대보다 설치 전환율 높음

## 3. 메시지를 받는 경험 — 어떻게 보여야 하나

### 3-1. "친구 추천" 카드와 일반 카드의 시각적 분리

- 일반 알고리즘 카드와 명확히 다른 레이어로 노출
- 카드 상단에: "OO이가 보냈어 💌" + 이모지/태그/한 줄 메시지 먼저 보임
- 그 아래: 프로필 열기 버튼
- 레퍼런스: Tinder Matchmaker "friend recommended" 태그, Cerca 공통 친구 최상단 노출
- PO 포인트: 공통 지인 소개 → 응답률 70% 상승 (LinkedIn 연구). 이 태그 하나가 UX의 핵심

### 3-2. 메시지 먼저, 프로필 나중

- 프로필 사진보다 친구 메시지를 먼저 읽게 되는 순서
- 순서: 친구 이름 + 메시지 → "프로필 열기" → 사진/정보
- 레퍼런스: S'More (사진 블러 처리, 대화가 먼저) — 여성 리텐션 개선 효과, Wingman 추천사 최상단
- PO 포인트: 외모보다 맥락을 먼저 보는 설계 → 매칭 품질 상승, 여성 유저 리텐션에 긍정적

### 3-3. 수락 전 "나중에 볼게" 옵션

- 즉시 수락/거절 강제하지 않음
- "오늘 저녁에 볼게" 저장 → 리마인더 알림
- 레퍼런스: Hinge (즉각 반응 강요 안 함), Bumble의 반대 사례 (24시간 강제가 여성 피로도 높임)
- PO 포인트: "지금 연애할 마음이 없어도 앱을 지우지 않는 이유" 설계

### 3-4. 거절해도 친구한테 안 알려짐

- 소개받은 카드를 거절해도 소개해준 친구에게 알림 없음
- 소개해준 사람은 "수락됨" 알림만 받음 (거절은 미통보)
- 레퍼런스: LinkedIn 소개 거절 시 조용히 처리
- PO 포인트: 거절 비용이 낮아야 수락 비용도 낮아짐. 양쪽의 심리적 안전망

## 4. 프레이밍 — 어떤 단어를 써야 하나

### 4-1. 버튼 레이블이 행동을 결정한다

| 단어 | 심리적 무게 | 적합한 맥락 |
|---|---|---|
| 추천하기 | 높음 — 공식적, 책임감 | ❌ 피해야 함 |
| 소개하기 | 중간 — 중립적 | △ 상황에 따라 |
| 이 사람 어때? | 낮음 — 질문형, 부담 없음 | ✅ 세렌디피티형 |
| 생각나는 친구 있어? | 낮음 — 탐색 유도 | ✅ 능동형 진입 |
| 친구한테 보내기 | 가장 낮음 — 카카오 공유와 동일 | ✅ 즉시 공유 |

- 레퍼런스: Kahneman/Tversky 프레이밍 효과 — 같은 행동도 표현 방식에 따라 참여율이 다름
- PO 포인트: "소개해주기" vs "친구한테 보내기" — 후자가 책임감이 낮아 전환율 높을 것

### 4-2. 알림 문구도 설계 대상

| 상황 | 추천 문구 |
|---|---|
| 친구가 카드 보냄 | "OO이가 너 생각하면서 보냈어 💌" |
| 친구가 수락함 | "네가 소개해준 사람, OO이가 관심 있어해!" |
| 매칭 성사 | "OO와 XX가 연결됐어. 네 덕분이야 🎉" |
| 비유저에게 초대 | "OO이가 너를 내친소에서 만나봤으면 해" |

## 5. 보상 — 메시지를 보낼 동기를 어떻게 만드나

### 5-1. 보상 지급 타이밍

- 전송 즉시: 씬구점 소량 (행동 강화)
- 상대방 수락 시: 씬구점 추가
- 매칭 성사 시: 씬구점 보너스 + 뱃지
- 레퍼런스: Dropbox (전환 완료 시점에 양쪽 모두 보상), Uber 리퍼럴 ROI 12배

### 5-2. 비금전적 보상

- "주선 고수" 뱃지 — 프로필에 노출
- 소개 성공률 통계 — "나는 지금까지 3커플 연결에 기여했어"
- 친구들의 감사 반응 — 매칭 성사 시 당사자가 주선자에게 "고마워" 한 탭 반응 가능
- 레퍼런스: Wingman 리더보드, Duolingo 스트릭 (누적 기록이 지속 동기를 만듦)

### 5-3. 소개받는 사람도 보상

- 친구 추천 카드를 수락하면 씬구점 (능동적 참여 유도)
- "친구 추천으로 연결된 사람과의 대화" 우선 노출
- 레퍼런스: Uber 리퍼럴 양면 보상 구조

## 6. 단계별 마찰 제거 — 어디서 이탈하나

```
[메시지 전송 funnel]

카드 발견
  → 공유 버튼 탭       ← 버튼이 눈에 잘 보여야 함
    → 친구 선택        ← 최근 대화 친구 상단 노출
      → 메시지 선택    ← 이모지/태그/프리셋 — 텍스트 입력 강제 금지
        → 전송         ← 화면 이탈 없이 완료
          → 완료 피드백 ← "OO이에게 보냈어! 씬구점 +5" 즉시 노출
```

- 레퍼런스: Airbnb Wishlist 공유 실패 사례 — 기능이 메뉴 안에 숨어 있어서 9%만 인지
- PO 포인트: 공유 버튼은 카드 위에 항상 노출. 메뉴 안에 넣는 순간 사용률 90% 소실

## 7. 타 앱 레퍼런스 요약

| 앱 | 참조할 포인트 | 적용 아이디어 |
|---|---|---|
| TikTok DM 공유 | 메시지 입력 선택 사항 — 빈 칸 전송 가능 | 이모지만으로도 전송 허용 |
| iMessage Tapback | 1탭 반응으로 충분한 소통 | 카드에 이모지 반응 버튼 |
| BeReal RealMoji | 좋아요보다 개인적, 댓글보다 쉬운 중간 포맷 | 사진 반응 소개 (고도화) |
| Letterboxd | 별점만 남겨도 유효한 리뷰로 인정 | 태그만 선택해도 완성된 소개 |
| Spotify 음악 공유 | 공유 자체가 취향 표현이 됨 | 카드 공유 = "내 안목" 표현 |
| AngelList 소개 | 템플릿 자동완성 → 응답률 50% | 프리셋 문장 선택 |
| Tinder Matchmaker | 친구 추천 태그 + 비유저도 참여 가능 | 카카오 딥링크 비유저 초대 |
| Duolingo Friend Streak | 연결된 유저 22% 더 많이 씀 | 소개 보낸 친구와 연결감 강화 |
| Wingman 리더보드 | 주선 횟수 누적이 정체성이 됨 | 주선 고수 뱃지 + 성공률 통계 |
| LinkedIn 소개 | 공통 친구 이름이 행동 트리거 | 카드에 "공통 친구 OO이" 표시 |
