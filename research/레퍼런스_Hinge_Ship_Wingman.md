# 데이팅앱 "친구 추천/주선" 기능 심층 리서치

---

## 1. Hinge Matchmaker

### 출시 배경 및 타임라인
- **2017년 9월**: Hinge Matchmaker라는 **별도 독립 앱**(standalone app)으로 iOS 베타 출시
- 핵심 컨셉: Hinge 유저가 아닌 사람도 Facebook 친구 목록에서 Hinge 가입자 두 명을 골라 매칭 제안 가능
- 매치메이커가 아이스브레이커 메시지까지 대신 보낼 수 있었고, 특정 친구의 프로필을 "잠금(Lock)"하여 집중 추천 가능
- 프라이버시 옵트아웃 기능 내장 (매치메이커에게 노출 거부 가능)
- 이후 독립 앱은 조용히 사라졌고, 현재는 Hinge 메인 앱 내에서 프로필 공유(Share Profile) 기능으로 흡수된 형태

### "Designed to be Deleted" 브랜딩과 성장 지표
- 이 슬로건은 Hinge의 핵심 마케팅 전략으로, "진지한 관계를 만들어 앱을 삭제하게 만든다"는 포지셔닝
- **매출**: 2018년 $8M → 2024년 $550M (전년 대비 38% 성장). Match Group은 Hinge를 **$1B 비즈니스**로 키울 수 있다고 공언
- **유저**: 총 3,000만 명, 유료 구독자 153만 명 (2년간 40만 명 순증)
- **월간 활성 유저**: 2023년 950만 → 2025년 1,100만 이상
- **Gen Z 비율**: 전체의 약 56% (17% 증가)
- 미국 데이팅앱 시장 점유율 3위 (Tinder, Bumble 다음), 영국/아일랜드/북유럽 일부 국가에서는 1위

### 프로필 프롬프트 시스템의 영향 (핵심 수치)
- **텍스트 프롬프트에 좋아요 → 실제 데이트로 이어질 확률이 사진 좋아요 대비 47% 높음** (2024년 데이터)
- 비디오 프롬프트가 있는 프로필은 인게이지먼트 50% 증가
- 음성 메모 대화는 실제 데이트로 이어질 확률 41% 증가
- 좋아요에 코멘트를 함께 보내면 매치율 40% 부스트
- 24시간 내 응답 시 데이트 확률 72% 증가
- 단, 유저의 63%가 프롬프트에 뭘 써야 할지 모르겠다고 응답 → 이 문제 해결이 핵심

### 2024-2025 최근 전략 변화
- **Prompt Feedback** (2025.1): OpenAI GPT-4o mini 기반, 프롬프트 답변이 너무 뻔한지 AI가 평가 후 코칭
- **Convo Starters** (2025): 프로필 사진/프롬프트 기반으로 AI가 3개의 대화 시작 팁 제안 (자동 답변이 아닌 "넛지" 방식)
- **Are You Sure?**: 공격적 메시지 전송 전 재고 유도
- **Comment Filter**: AI 기반 불쾌 메시지 자동 필터링
- Match Group이 2025년 AI 혁신에 **$20M~$30M 투자** 계획
- Hinge 창업자 Justin McLeod는 2025년 12월 CEO 사임 후 **Overtone**이라는 AI-first 데이팅앱 신규 창업

### 내친소 적용 인사이트 3가지

1. **프롬프트 기반 추천서 시스템 도입**: Hinge 데이터에서 텍스트 프롬프트가 사진보다 47% 더 데이트로 이어진다. 내친소의 친구 추천서도 "이 친구의 매력 포인트 3가지" 같은 구조화된 프롬프트로 작성하게 하면, 단순 칭찬보다 대화 시작률이 크게 높아질 것.

2. **AI 코칭으로 추천서 품질 관리**: Hinge의 Prompt Feedback처럼, 추천서 작성 시 "너무 뻔한 표현입니다" 같은 피드백을 주면 63%가 뭘 써야 할지 모르는 문제를 해결 가능. 추천서 작성 허들을 낮추면서도 품질을 높이는 핵심 장치.

3. **"삭제되도록 설계" 포지셔닝 차용**: Hinge의 $550M 매출은 역설적으로 "앱을 지우게 만든다"는 메시지에서 나왔다. 내친소도 "친구가 주선해서 만났으니 앱이 필요 없어진다"는 스토리를 브랜딩에 활용하면, 진지한 유저 유입에 효과적.

## 2. Ship (Match Group x Betches Media, 2019~2022)

### 정확한 타임라인
- **2019년 1월 22일**: iOS 출시 (Match Group + Betches Media 파트너십)
- **2019년 8월**: Android 출시, 이 시점에서 Match Group 패밀리 내 **가장 빠르게 성장하는 데이팅앱**으로 평가
- **핵심 수치**: 전체 매치의 53%가 친구 대리 스와이프에서 발생
- **2022년 9월 30일**: 서비스 종료. Match Group이 조용히 앱 중단

### 실패 원인 분석

**구조적 문제:**
- Match Group CEO Bernard Kim의 실적 발표 발언: "성장하는 브랜드에 투자를 집중하고, 성장을 견인하지 못하는 영역에서는 철수해야 한다" → Ship이 충분한 규모 성장을 이루지 못함
- 독립 앱이 아닌 Match Group의 포트폴리오 내 실험작으로, Tinder/Hinge에 비해 투자 우선순위 낮음

**Crew 기능의 UX 문제점:**
- 크루 멤버의 이름/사진이 보이지 않는 UI 혼란
- 다수의 친구 그룹(직장 동료, 대학 친구 등)을 분리된 크루로 만들 수 없어 하나의 크루에 모든 친구를 합쳐야 함
- 프로필에 사진, 키, 직업, 학교, 거리 정도만 표시 → 친구가 대리 판단하기에 정보 부족
- 그룹 채팅 메시지 전달 지연 (글리치)
- 매치 알림 미수신 버그
- 같은 장소에 있는 크루 멤버끼리도 다른 후보를 보는 싱크 문제

**시장 문제:**
- Betches Media의 인스타 팔로워 600만 명이라는 도달 범위에도 불구하고, 실제 데이팅앱 전환율은 낮았음
- 유저 풀이 충분하지 않아 매칭 가능한 후보 자체가 적었다는 리뷰 다수

### Match Group이 Ship에서 배운 교훈
- 53%가 친구 대리 스와이프 매치라는 데이터는 "친구 참여" 컨셉 자체의 수요를 검증
- 이후 **Tinder, Hinge, Bumble 모두 프로필 공유 기능** 도입
- Tinder는 **Double Date** 기능 출시 (친구와 함께 다른 친구 쌍과 매칭)
- 즉, "독립 앱"이 아닌 "기존 앱의 부가 기능"으로 통합하는 것이 정답이었음

### 내친소 적용 인사이트 3가지

1. **다중 크루(그룹) 지원 필수**: Ship 실패의 핵심 UX 문제 중 하나가 친구 그룹을 분리하지 못한 것. 내친소에서는 "대학 친구", "직장 동료" 등 별도 주선 그룹을 만들 수 있게 해야 한다. 맥락이 다른 친구들이 같은 공간에 섞이면 추천 품질이 떨어진다.

2. **풍부한 프로필 정보 제공**: Ship은 사진+기본 스펙만 보여줘서 친구가 판단할 근거가 부족했다. 내친소는 친구가 대리 판단할 수 있도록 가치관, 관심사, 데이트 스타일 등 풍부한 정보를 노출해야 함. 특히 주선자 전용 뷰를 별도 설계하는 것이 핵심.

3. **독립 앱이 아닌 통합 기능으로**: Ship의 가장 큰 교훈은 "친구 추천을 별도 앱으로 분리하면 유저 풀이 쪼개져 양쪽 다 위축된다"는 것. 내친소는 싱글 유저와 주선자가 같은 앱 안에 공존하되, 역할에 따라 UI가 전환되는 구조가 유리하다.

## 3. Wingman (2018~현재)

### 출시 및 현황
- **2017년 4월**: 초기 런칭 (PR 기준)
- **2018년 1월**: 미국 시장 정식 진출
- 창업자: **Tina Wilson** (인테리어 디자인/프로젝트 매니지먼트 출신, 10년 연애 후 갑자기 싱글이 되며 창업 동기)
- 주요 시장: 영국, 미국 → 글로벌 확장 전략 진행
- 구체적 유저 수는 비공개이나, 초기 리뷰에서 "유저 풀이 아직 작다"는 지적이 있었고, 니치 마켓으로 꾸준히 성장 중
- 여성 유저(싱글+윙우먼)가 남성보다 약간 많으나 시간이 갈수록 비율 균형화

### Single 모드 vs Wingman 모드
- 앱 핵심 구조: 싱글은 프로필을 등록하되, 직접 스와이프하지 않음. **윙맨/윙우먼이 프로필을 만들고 스와이프**하는 것이 기본
- **윙맨이 있는 싱글은 매치율 80% 높음** (앱 공식 데이터)
- 성공적으로 매칭된 싱글이 다시 윙맨으로 전환하는 선순환 구조 존재
- 윙맨이 좋은 결과를 경험하면 다른 친구에게도 확장하는 바이럴 패턴

### 리더보드/게이미피케이션
- 친구들이 만든 매치 수로 점수가 올라가는 **리더보드** 시스템
- 여러 친구를 윙맨/윙우먼으로 초대 가능 → 누가 더 많은 매치를 만드는지 경쟁
- 이 게이미피케이션이 "주선 행위 자체를 재미있게" 만드는 핵심 장치
- 2019년에 윙맨 보상 시스템(reward system) 강화 계획 발표

### Ship과 달리 생존한 핵심 이유

| 요인 | Ship | Wingman |
|------|------|---------|
| 앱 구조 | 싱글이 직접 스와이프 + 친구가 보조 (하이브리드) | 윙맨이 주도적으로 매칭 (친구 역할 명확) |
| 역할 정의 | 누가 주인공인지 모호 | 싱글=수동, 윙맨=능동으로 명확 |
| 대기업 의존 | Match Group 포트폴리오 내 실험작 → 우선순위 밀림 | 독립 스타트업 → 생존 자체가 미션 |
| 수익 모델 | Match Group 전체 매출에 묻힘 | 독립적으로 수익화 추구 |
| 바이럴 루프 | 크루 초대가 번거로움 | 윙맨이 성공 → 다른 친구에게 확장하는 자연스러운 루프 |
| 프라이버시 | 친구 채팅에서 모든 게 공유 | 윙맨이 선별해서 전달하는 구조 |

### 내친소 적용 인사이트 3가지

1. **주선자 역할을 명확하게, 그리고 재미있게**: Wingman의 핵심 생존 요인은 "친구가 주도적으로 매칭한다"는 역할 명확성. 내친소도 주선자가 단순히 추천서만 쓰는 게 아니라, 실제로 후보를 탐색하고 제안하는 "능동적 주선" 모드를 제공하면 참여도가 높아진다.

2. **리더보드로 주선자 간 경쟁 유도**: Wingman의 리더보드는 주선 행위를 게임처럼 만든다. 내친소에서도 "이번 달 최고의 주선왕" 같은 경쟁 요소를 넣으면, 주선자 리텐션 문제(한 번 추천하고 이탈)를 해결할 수 있다. 특히 내친소의 "썬구리" 보상 시스템과 결합하면 시너지 가능.

3. **싱글 → 윙맨 전환 루프 설계**: Wingman에서 매칭 성공한 싱글이 다시 윙맨으로 전환하는 패턴이 관찰됨. 내친소도 "만남이 성사된 유저 → 다른 친구 주선" 전환 흐름을 자연스럽게 유도하면, 유저 획득 비용 없이 주선자 풀이 자동 확장되는 선순환 가능.
