# 내친소 친구 추천 기능 — 액션 브레인스토밍

> 목적: 기능 개발 전 아이디어 확장 및 레퍼런스 정리
> 마지막 업데이트: 2026-03-25

---

## 전제: 우리가 풀어야 할 구조

```
[지금]
주선자 → 할 수 있는 게 없음 → 이탈
소개받는 사람 → 알고리즘 매칭과 다를 게 없음 → 차별점 없음

[목표]
주선자 → 다양한 액션 가능 → 참여 지속
소개받는 사람 → "사람이 골라줬다"는 온기 → 신뢰 기반 수락
```

**핵심 전제: 추천사는 두 가지 맥락에서 발생한다**

| 구분 | 능동형 | 세렌디피티형 |
|---|---|---|
| 시작점 | 친구를 위해 상대를 찾으러 들어옴 | 내 피드 보다가 친구 생각남 |
| 동기 | 친구 중심 ("얘 좋은 사람 만났으면") | 상대 중심 ("이 사람 OO이랑 잘 맞겠다") |
| 추천사 무게 | 신중하고 정성스럽게 | 즉흥적, 한 줄이면 충분 |
| 소개받는 감정 | "나를 위해 일부러 찾아줬다" | "나 생각났대" 따뜻함 |
| 수락 심리 | 신뢰감 → 높은 기대 | 호기심 → 낮은 허들 |

## 주선자 관점 — 가능한 액션들

### A. 능동형 진입 ("친구한테 좋은 사람 찾아줄게")

**A-1. 친구 대신 피드 탐색**
`역할통합` `능동형` `데이팅×내가소개`

> "내 친구 OO이를 위해 피드를 본다"

- 친구를 선택하면 → 그 친구 프로필 기준으로 필터링된 피드가 열림
- "OO이라면 좋아할 것 같은 사람"을 탐색
- 마음에 드는 카드에 "소개하기" 탭

레퍼런스:
- **Ship (Match Group)**: crew가 당사자 대신 스와이프. 전체 매칭의 53%가 crew 스와이프로 발생
- **Wingman**: "내 친구를 위한 Wingman 모드" 별도 운영. 친구 취향 입력 후 대리 탐색

내친소 포인트:
- 친구 프로필의 "이런 사람 원해요"를 기반으로 피드 자동 필터링
- 주선자가 스와이프한 기록이 당사자에게 "OO이가 추천한 사람"으로 노출

---

**A-2. 정성 추천사 작성**
`능동형` `데이팅×내가소개`

> "이 친구가 어떤 사람인지 내가 직접 써줄게"

- 소개하는 친구의 프로필에 추천사 추가
- 항목: 성격 한 줄 / 추천하는 이유 / 어떤 사람 만나면 좋을지

레퍼런스:
- **Wingman**: "친구의 좋은 점 / 나쁜 점 / 특이한 점" 3가지 항목 작성. 친구가 쓴 추천사가 프로필 핵심 콘텐츠로 노출
- **LinkedIn 추천서**: 지인이 직접 써주는 신뢰 기반 텍스트. 받는 사람의 프로필에 상시 노출

내친소 포인트:
- 추천사는 **소개받는 사람도 볼 수 있는 공개 콘텐츠**
- 길수록 좋은 게 아니라 **진정성 있는 한 줄**이 핵심

---

**A-3. 양방향 동시 소개 (그룹 소개)**
`역할통합` `능동형` `데이팅×내가소개`

> "두 사람에게 동시에 연결 제안을 보낸다"

- 주선자가 A와 B 두 사람을 선택
- 두 사람 모두에게 동시에 "OO이가 너희를 연결해주고 싶어해" 알림
- 둘 다 수락하면 연결

레퍼런스:
- **Hinge Matchmaker (2017)**: 두 명의 Hinge 유저를 선택 → 양쪽에 동시 아이스브레이커 전달
- **LinkedIn "소개 요청"**: 2촌에게 소개 요청 시 공통 1촌이 중간에서 연결해주는 구조
- **카카오톡 단톡방 소개**: 오프라인에서 이미 일어나고 있는 행동 — 앱 안으로 가져오기

내친소 포인트:
- 한국 소개팅 문화에 가장 가까운 형태
- "주선자가 두 사람을 이어주는" 역할이 명확해짐

---

### B. 세렌디피티형 진입 ("보다가 친구 생각났어")

**B-1. 피드에서 즉시 공유**
`세렌디피티형` `데이팅×내가소개` `비데이팅×내가소개`

> "이 카드 보다가 OO이 생각났어, 바로 보내줄게"

- 피드의 모든 카드에 "친구에게 소개" 버튼
- 탭 → 친구 목록 → 선택 → 한 줄 메모 (선택) → 전송
- 전체 흐름이 3초 이내에 완료되어야 함

레퍼런스:
- **Hinge "Send Profile"**: 프로필을 친구에게 DM으로 전송하는 기능. "이 사람 어때?" 한마디와 함께
- **카카오톡 공유**: 이미 일어나고 있는 행동. 스크린샷 찍어서 단톡방에 올리는 것을 대체

내친소 포인트:
- 현재 사용자들이 이미 카카오톡으로 하고 있는 행동을 앱 안으로
- 공유하면 → 친구에게 알림 + 주선자에게 씬구점 적립

---

**B-2. 한 줄 메모 소개**
`세렌디피티형` `비데이팅×내가소개`

> "길게 쓸 필요 없어, 떠오른 한마디만"

- 추천사 작성 없이 태그/이모지/짧은 한 줄만으로도 전달 가능
- 예시: `"너 스타일이야 ㅋㅋ"` / `#유머있음 #카페좋아함`

레퍼런스:
- **Locket**: 친구에게 사진 공유할 때 텍스트 없이 이미지만 보내도 됨. 마찰 최소화
- **Instagram DM "이 게시물을 보냈습니다"**: 한마디 없이 공유만 해도 충분한 맥락 전달

내친소 포인트:
- 세렌디피티형은 **순간의 감정**이 동력 → 마찰이 1초라도 길면 그냥 넘김
- "한 줄도 많아. 태그만 골라" 수준까지 낮춰야 함

---

**B-3. "오늘 생각난 사람" 큐레이션**
`세렌디피티형` `역할통합` `비데이팅×내가소개`

> "오늘 피드 보다가 친구들 생각이 났어"

- 하루에 몇 명이나 주선자가 태그를 달았는지 요약
- "오늘 OO이한테 1명 소개 요청 보냄" 알림

레퍼런스:
- **Thursday**: 목요일 하루만 앱 활성화 → 희소성으로 참여율 상승
- **Locket**: 매일 베프들 사진이 자동으로 뜸 → 일상적 루틴화

내친소 포인트:
- 주선 행동을 **일상의 루틴**으로 만드는 것이 핵심
- "오늘 내 피드 보면서 친구 생각났어?" 라는 습관적 행동 유도

## 소개받는 사람 관점 — 가능한 액션들

### C. 소개 알림 수신

**C-1. "친구 추천" 전용 카드**
`역할통합` `데이팅×친구가소개`

> "알고리즘 카드와 다른, 별도 레이어로 노출"

- 일반 피드와 분리된 "친구가 소개해준 사람" 섹션
- 카드에 "OO이가 소개해줬어요" + 추천사 한 줄 노출
- 알고리즘 추천과 시각적으로 명확히 구분

레퍼런스:
- **Cerca**: 공통 친구 있는 카드를 피드 상단에 우선 노출
- **Hinge**: "Your Turn" 섹션으로 내가 액션해야 할 카드 별도 분류

내친소 포인트:
- 친구 추천 카드는 일반 매칭보다 수락률이 높을 것 → 먼저 노출이 UX 핵심

---

**C-2. 수락 전 "추천사 먼저 보기"**
`능동형` `세렌디피티형` `데이팅×친구가소개`

> "누군지 보기 전에, 친구가 뭐라고 했는지 먼저 읽어"

- 프로필 열기 전에 주선자의 추천사 텍스트가 먼저 보임
- "OO이가 이렇게 소개했어요: ___"
- 추천사가 없으면 → "OO이가 피드 보다가 너 생각이 났대요"

레퍼런스:
- **Wingman**: 친구가 쓴 소개글이 프로필 최상단에 배치
- **Netflix**: "OO님이 좋아할 것 같아요" — 추천 이유가 먼저

내친소 포인트:
- 추천사의 내용보다 **"사람이 골랐다"는 사실** 자체가 신뢰의 근거
- 짧은 추천사도 충분 — "걔 생각났어" 한마디가 제일 강력할 수 있음

---

**C-3. 나중에 볼게 (북마크)**
`역할통합` `능동형` `세렌디피티형`

> "지금 볼 시간이 없어, 나중에"

- 소개 요청을 즉시 수락/거절하지 않고 저장
- "오늘 저녁에 볼게" 리마인더 설정 가능

레퍼런스:
- **Hinge**: "We Met" 후속 질문으로 매칭 품질 추적. 즉각 반응 강제 안 함
- **Bumble**: 매칭 후 24시간 내 메시지 전송 의무 → 반대로 내친소는 압박 없음이 차별점

내친소 포인트:
- "지금 연애할 마음이 없어"도 앱을 안 지워도 되는 이유
- 나중에 볼게 저장 수 → 잠재 매칭 파이프라인 지표로 활용 가능

## 주선자 동기 유지 — 보상과 게이미피케이션

### D. 언제 보상을 줄 것인가

**문제:** 성사됐을 때만 보상하면 주선자가 지속하지 않는다.
Ship이 망한 이유와 정확히 같다.

```
소개 요청 전송  →  씬구점 X점 (즉시 지급)
상대방 수락     →  씬구점 Y점
대화 시작       →  씬구점 Z점
매칭 성사       →  주선자 뱃지 + 씬구점 보너스
```

레퍼런스:
- **Dropbox 리퍼럴**: 초대 링크 클릭 시점이 아니라 **가입 완료 시** 양쪽 모두 보상. 전환 행동에 보상 연결
- **Uber 리퍼럴**: 드라이버와 승객 모두 보상 (양면) → ROI 12배
- **Wingman 리더보드**: 누적 소개 성사 수 기반 랭킹 → "주선 고수" 타이틀

내친소 포인트:
- 씬구점의 핵심 가치 = **프로필 열기** → 성공적인 소개를 많이 한 사람이 더 많은 프로필을 볼 수 있음
- 보상이 앱의 핵심 가치와 연결될수록 이탈 후 재진입 동기가 생김

---

**D-1. 주선 고수 뱃지**
`역할통합` `능동형`

> "이 앱에서 소개를 가장 많이 성사시킨 사람"

- 누적 소개 성사 수 기반 타이틀
- 프로필에 노출 → "주선 잘하는 친구"라는 정체성 형성

레퍼런스:
- **Wingman 리더보드**: 소개 성사 수로 순위 결정, 매주 리셋
- **Reddit 카르마**: 누적 기여도가 프로필에 상시 노출

---

**D-2. 소개 성공률 통계**
`역할통합` `능동형`

> "내가 소개한 사람 중 몇 명이나 연결됐어?"

- "OO님의 소개 성공률: 68%" 개인 통계
- "지금까지 3커플 연결에 기여했어요"

레퍼런스:
- **Hinge "We Met" 리포트**: 앱이 실제 만남으로 이어졌는지 트래킹
- **Duolingo 스트릭**: 누적 기록이 지속 동기를 만듦

## 앱 밖 바이럴 — CAC를 낮추는 액션들

### E. 외부 유입 루프 설계

**E-1. 카카오 공유로 프로필 미리보기**
`세렌디피티형` `비데이팅×내가소개`

> "비유저 친구한테도 소개할 수 있어야 한다"

```
주선자가 피드에서 카드 발견
→ "카카오로 공유" 선택
→ 친구에게 "이 사람 어때?" 메시지 전송
→ 링크 열면 → 프로필 일부 미리보기 (흐림 처리)
→ "전체 보기 위해 내친소 앱 설치" CTA
→ 설치 후 주선자와 자동 연결
```

레퍼런스:
- **Locket**: 앱 없이는 사진을 볼 수 없음. 친구가 공유하면 자동으로 설치 유도
- **Hinge Matchmaker**: 비유저 친구를 초대해서 매칭 → 설치 후 해당 카드 먼저 노출

내친소 포인트:
- 지금 카카오톡으로 하는 행동 (스크린샷 공유)을 공식 기능으로 대체
- 미리보기 → 설치 → 연결이 한 흐름으로 완성되어야 함

---

**E-2. "너 소개받기 기다리는 사람 있어" 초대**
`비데이팅×친구가소개` `세렌디피티형`

> "이미 누군가가 너를 소개하고 싶어해"

```
주선자가 비유저 친구를 소개하고 싶음
→ 친구 연락처로 "소개 대기 중" 상태 생성
→ 친구에게 카카오 메시지: "OO이가 너를 내친소에서 소개해주고 싶어해"
→ 설치 → 소개 카드 확인
```

레퍼런스:
- **Hinge Matchmaker**: 비유저를 초대해서 매칭 연결
- **Facebook "친구가 태그했어요"**: 이미 무언가가 기다리고 있다는 알림이 설치 전환율 극대화

내친소 포인트:
- 단순 초대보다 **"이미 뭔가 기다리고 있어"** 프레이밍이 훨씬 강력
- Locket이 "친구가 있어야 앱이 작동한다"로 2주 200만 가입 달성한 원리와 동일

## 한국 문화 특화 액션

### F. 오프라인 소개팅 문화를 디지털화

**F-1. 번개 소개 모임**
`역할통합` `비데이팅×내가소개`

> "앱에서 소개받은 사람들이 실제로 만나는 자리"

- "이번 주말 강남 XX카페에서 소개팅 번개" 이벤트
- 주선자가 자신이 소개한 커플을 이벤트에 초대
- 주선자도 참석 가능 → 분위기 메이커 역할

레퍼런스:
- **Thursday**: 목요일 하루만 앱 활성화 + 오프라인 이벤트 연동. 90만 다운로드
- **Bumble BFF + Geneva 인수**: 그룹/커뮤니티 기반으로 전환

---

**F-2. 단톡방 맥락 소개**
`역할통합` `비데이팅×내가소개` `비데이팅×친구가소개`

> "카카오 단톡방에서 일어나는 소개팅을 앱 안으로"

- 그룹 기능: 3~5명의 친구들이 하나의 "소개 그룹" 구성
- 그룹 안에서 "이 사람 어때?" 피드백을 함께 논의
- 최종 결정은 당사자가

레퍼런스:
- **Ship**: crew 그룹이 대리 스와이프. 단체로 의사결정하는 구조
- **카카오톡 단톡방 소개팅**: 이미 한국에서 일어나고 있는 행동 패턴

[보상/게이미피케이션]

  ├── J-1. 소개 그룹 3~5인 (Ship crew → 보상 보완)

## G. 추천사 = 프로필 자산 (누적되는 신뢰)

> "소개 메시지가 사라지지 않고, 내 프로필에 쌓인다"

지금 구조에서 추천사는 일회성 메시지다. 보내면 끝.
하지만 추천사가 프로필에 남으면 완전히 다른 가치가 생긴다.

**G-1. 친구 추천사 프로필 노출**
`능동형` `데이팅×친구가소개`

- 친구들이 나에 대해 써준 한 줄들이 프로필 하단에 누적 노출
- 소개받는 사람이 프로필 열면 → "친구 3명이 이렇게 소개했어요" 섹션 존재
- 추천사가 많을수록 → 더 신뢰할 수 있는 프로필

레퍼런스:
- **LinkedIn 추천서**: 지인이 직접 써준 텍스트가 프로필 상시 노출 → B2B에서 가장 신뢰받는 신호
- **Wingman**: 친구가 쓴 소개글이 프로필 최상단에 배치 → 알고리즘 매칭과 명확히 다른 감정
- **Airbnb 호스트 리뷰**: 누적된 후기가 곧 신뢰 자산

내친소 포인트:
- 추천사를 많이 받은 사람 → 더 매력적인 프로필 → 쓰여지고 싶다는 동기 생김
- 추천사 없는 프로필 vs 있는 프로필 → 수락률 차이가 클 것 → 유저가 스스로 친구에게 추천사 요청하게 됨
- "내 추천사 받기" 버튼이 이미 마이페이지에 존재 → 이 버튼의 결과물이 프로필에 노출되어야 함

**G-2. 추천사 요청하기**
`능동형` `비데이팅×친구가소개`

- 내가 친구에게 "나에 대해 한 줄 써줄 수 있어?" 요청 가능
- 친구는 앱 설치 없이 링크만으로 짧게 작성 가능 (비유저 참여 허용)
- 작성된 추천사는 내 프로필에 노출 + 친구에게 "OO이가 고마워해요" 알림

레퍼런스:
- **LinkedIn "추천 요청"**: 비연결 상태에서도 추천 요청 메시지 전송 가능
- **Google 리뷰 요청 링크**: 링크 하나로 리뷰 작성 페이지 이동. 앱 불필요

## H. 네트워크 코드 연계 (입장 코드로서의 추천)

> "친구 추천 = 이 네트워크에 들어올 자격"

Notion 유입 매트릭스에서 "네트워킹 코드 받아서 들어간 거야"가 강력한 유입 명분으로 분류된 바 있다.
친구 추천/주선 시스템이 이 역할을 자연스럽게 할 수 있다.

**H-1. 추천이 곧 입장 코드**
`비데이팅×친구가소개` `역할통합`

- 친구 추천 없이는 가입이 어렵거나 노출이 제한됨
- "아무나 들어오는 앱이 아니야" → 희소성 + 안전감 동시에
- 추천해준 친구 = 보증인 → 책임 구조가 생김

레퍼런스:
- **Clubhouse 2021**: 초대장 2장만 지급. 초대장이 eBay에서 거래됨 → 희소성이 바이럴 자체
- **내친소 Notion 매트릭스**: "심사통과제 (이펙트 🔴 상)" — 프라이빗 코드 개념이 이미 팀 내 논의됨
- **Discord 서버 초대**: 초대 링크 없이는 입장 불가 → 초대한 사람이 책임지는 구조

내친소 포인트:
- 현재 "친구 초대 시 씬구점" 구조 → "친구 추천 없으면 기본 노출 제한" 으로 강화 가능
- 추천을 보내는 행위 = "이 사람을 우리 네트워크에 초대한다"는 의미로 격상
- 여성 유저 입장: "여기는 아무나 없어, 친구 추천 받은 사람들이야" → 안심 명분

**H-2. 취향 네트워크 코드**
`비데이팅×내가소개` `능동형`

- 단순 친구 추천이 아니라 **취향 커뮤니티 기반 추천**
- "갓생 모임", "러닝 진심인 사람들" 같은 취향 그룹 안에서 주선이 발생
- 같은 취향 그룹 안의 추천 = 더 높은 신뢰 신호

레퍼런스:
- **Tapple (일본)**: 관심사 그룹 입장 → 그룹 안에서 매칭. 같은 취향끼리라는 안전감
- **Notion 매트릭스 Tier 1**: 갓생/자기관리/임장 등 취향 그룹이 강력한 유입 명분

## I. 주선 = 유입 명분 (기능이 마케팅이 됨)

> "주선하러 깐 거야" 라는 말이 자연스럽게 나와야 한다

Notion 팀 회의에서 "주선하러 깐 거야"는 Quick Win 2위 (이펙트 🔴 상 / 난이도 🟢 하)로 평가됐다.
지금은 카피로만 쓸 수 있지만, 주선 기능이 강력해지면 이 말이 진짜 사실이 된다.

**I-1. 기능 개편이 광고 카피가 되는 구조**
`역할통합` `비데이팅×내가소개`

```
지금: "주선하러 깐 거야" → 카피는 있지만 실제 기능이 약함 → 허언이 됨
목표: "주선하러 깐 거야" → 실제로 피드에서 친구한테 소개 보낼 수 있음 → 진짜가 됨
```

- 주선 기능이 충분히 작동할 때 → "주선하러 왔어"라는 말이 자연스럽게 확산
- 데이팅앱 다운로드 허들 ("내가 이런 앱까지…") 을 "주선해주러 들어간 것"으로 프레이밍 전환

레퍼런스:
- **Bumble BFF**: "친구 사귀러 왔어" → 데이팅앱 낙인 없이 가입 가능
- **위피**: "소셜앱"으로 포지셔닝 → 데이팅 거부감 있는 유저도 유입

**I-2. 주선자 스토리 바이럴**
`세렌디피티형` `비데이팅×내가소개`

- 주선 성사 시 → "OO이가 두 사람을 연결했어요 🎉" 공유 가능한 카드 생성
- SNS 공유 → "나 내친소에서 친구 소개팅 성사시켰어 ㅋㅋ" → 자연스러운 바이럴
- 주선자의 성공 스토리가 UGC 콘텐츠가 됨

레퍼런스:
- **Duolingo 스트릭 공유**: 100일 달성 카드 → SNS 자동 공유 → 바이럴
- **Strava 세그먼트 1위**: 달성 순간 공유 욕구 발생

## J. 그룹 주선 (커뮤니티 단위 참여)

> "혼자 소개하는 게 아니라, 우리가 함께 고른다"

개인이 1:1로 소개하는 구조를 넘어서, 그룹이 함께 주선에 참여하는 구조.
한국 소개팅 문화에서 단톡방 소개가 이미 이 방식이다.

**J-1. 소개 그룹 (3~5인)**
`역할통합` `비데이팅×내가소개`

- 친한 친구들로 구성된 소개 그룹 생성
- 그룹 안에서 "이 사람 어때?" 카드를 함께 논의
- 다수결 or 당사자 최종 결정

레퍼런스:
- **Ship crew**: 그룹이 함께 대리 스와이프하는 구조. 단, 그룹 리텐션 실패 → 보상 설계 필수
- **카카오톡 단톡방 소개팅**: 이미 한국에서 가장 일반적인 소개 방식

**J-2. 취향 커뮤니티 내 주선**
`능동형` `비데이팅×내가소개`

- Notion Tier 1: 갓생/자기관리/임장 등 취향 그룹 → 그룹 안에서 자연스럽게 주선 발생
- "갓생 모임 안에 있는 친구한테 괜찮은 사람 소개해줄게"
- 취향 공동체 = 신뢰 공동체 → 같은 그룹 안의 소개는 더 높은 수락률

레퍼런스:
- **Tapple (일본, 23M)**: 관심사 그룹 입장 후 매칭. 같은 취향 = 기본 신뢰
- **스카이피플**: 직장인 커뮤니티 → 데이팅은 자연 발생
