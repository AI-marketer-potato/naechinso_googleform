# PO 스펙 결정 근거 모음

> 코어 루프 MVP 개발 시 결정이 필요한 항목별로, 레퍼런스에서 확인된 수치·사례를 정리.
> "왜 이렇게 해야 하나?"에 대한 답을 내리기 위한 자료.

## 1. 역할 통합: 피드 하나 vs 탭 분리

**결정할 것:** 주선자/소개받는 사람 역할을 하나의 피드에 넣을지, 탭으로 나눌지.

| 앱 | 구조 | 결과 |
|---|---|---|
| Ship (Match Group) | crew 역할 분리 — 주선자는 별도 화면에서 대리 스와이프 | 매칭의 53%가 crew에서 발생했으나, **주선자 이탈로 2022년 서비스 종료** |
| Wingman | 같은 앱에서 "나도 매칭 + 친구도 매칭" 전환 가능 | **생존 중**. "Wingman 있으면 매칭 80% 증가" 공식 수치 |
| Hinge Matchmaker (2017) | 별도 앱으로 출시 → 나중에 본앱 기능으로 통합 | 별도 앱일 때 DAU 부족 → 통합 후에야 작동 |

**판단 근거:**
- Ship이 매칭 자체는 잘 됐는데(53%) 죽은 이유 = 주선자가 자기 매칭은 못 하니까 앱을 켤 이유가 사라짐
- Wingman이 살아남은 이유 = "나도 쓰면서 친구도 소개" 가 한 화면
- **Ship의 실패 원인이 UX가 아니라 구조였다는 점이 핵심.** crew 화면 자체는 문제없었는데, 주선자에게 self-serving 이유가 없었음

**→ 스펙 방향:** 피드 하나에 통합. 같은 카드를 보면서 "내가 좋아요" 또는 "친구에게 소개" 두 액션 모두 가능하게.

## 2. "친구에게 소개" 버튼 위치와 라벨

**결정할 것:** 카드에서 소개 버튼을 어디에 놓고, 뭐라고 쓸지.

| 레퍼런스 | 공유 버튼 위치 | 라벨 | 효과 |
|---|---|---|---|
| TikTok | 영상 오른쪽 사이드바 (항상 노출) | 공유 아이콘 (텍스트 없음) | 공유율 업계 최고. 아이콘만으로 충분히 인지 |
| Hinge "Send Profile" | 프로필 우측 상단 점 3개 메뉴 안 | "Send to a Friend" | 접근성 낮음 → 실사용 적음 (숨겨진 기능 취급) |
| Instagram Reels | 콘텐츠 하단 DM 전송 아이콘 | 비행기 아이콘 | TikTok과 유사한 위치, 높은 공유율 |

**판단 근거:**
- 메뉴 안에 숨기면 안 됨 (Hinge 사례 — 기능은 있는데 안 씀)
- 항상 보이는 위치에 있어야 행동이 발생함 (TikTok/Instagram)
- 라벨은 한국어로 자연스럽게: "이 사람 어때?" (카카오톡 소개팅 때 실제로 쓰는 말)

**→ 스펙 방향:**
- 위치: 카드 하단 또는 좋아요 버튼 옆 (항상 노출)
- 라벨: "이 사람 어때?" — 추천하기/소개하기보다 일상적인 톤
- 점 3개 메뉴 안에 넣으면 안 됨

## 3. 추천 메시지 포맷 (MVP)

**결정할 것:** 친구에게 소개할 때 붙이는 메시지를 어떤 형태로 받을지.

| 포맷 | 레퍼런스 | 장점 | 단점 |
|---|---|---|---|
| 이모지만 (💘👀🔥) | Instagram Quick React | 마찰 최소 (탭 1번) | 정보량 0 — 왜 소개하는지 모름 |
| 태그 선택 (#네스타일 #취향저격) | TikTok 해시태그 | 마찰 낮음 + 맥락 전달 | 태그 목록 설계 필요 |
| 30자 자유 텍스트 | KakaoTalk 소개팅 실제 패턴 | 자연스러움 | Hinge 데이터: 63%가 뭘 써야 할지 모름 |
| 프리셋 문장 (편집 가능) | AngelList 소개 템플릿 | 응답률 50%. 빈 화면 공포 제거 | 복붙 느낌 → 진정성 의문 |
| 음성 15초 | 카카오톡 음성메시지 | 감정 전달력 최고 | 개발 공수 큼, MVP에 과함 |

**핵심 수치:**
- Hinge: 텍스트 프롬프트에서 시작된 대화가 사진 좋아요보다 **첫 데이트 전환율 47% 높음**
- 그런데 유저의 63%가 프롬프트에 뭘 써야 할지 모름 → 빈칸 주면 대부분 안 씀
- AngelList: 프리셋 소개 템플릿 제공 시 응답률 50%

**→ 스펙 방향 (MVP):**
- **프리셋 문장 + 편집 가능** 조합이 가장 유력
- 예: "얘 진짜 좋은 사람이야, [한 줄 추가]" 형태로 기본 문장 제공 + 수정 가능
- 이모지 리액션은 보조로 추가 가능하나, 단독으로는 약함
- 음성 메시지는 신뢰+확산 이후 검토

## 4. 동의 구조: 소개 전 동의 vs 사후 알림

**결정할 것:** 내 프로필이 친구에 의해 공유될 때, 사전에 동의를 받을지 / 사후에 알려줄지.

| 앱 | 방식 | 반응 |
|---|---|---|
| Hinge Matchmaker (2017) | 동의 없이 소셜 그래프 노출 (opt-out) | **"creepy"** 반응 → 별도 앱 실패의 직접 원인 |
| LinkedIn 상호 소개 | 양쪽 모두에게 "소개 받을래요?" 먼저 물음 (opt-in) | 수용됨. B2B 문맥이지만 구조는 동일 |
| Cerca | 연락처 전체 동기화 (opt-in이지만 범위가 넓음) | 프라이버시 우려 존재 — 아직 초기라 이슈 미폭발 |

**판단 근거:**
- Hinge Matchmaker가 2017년에 실패한 가장 큰 이유가 동의 구조
- "내 친구가 나도 모르게 내 프로필을 남에게 보여줬다" = 신뢰 파괴
- 반대로, "OO이 너를 누군가에게 소개하고 싶대. 볼래?" = 기대감 + 존중

**→ 스펙 방향:**
- **소개 전 수신자 동의 필수** (opt-in)
- 플로우: 주선자가 소개 버튼 탭 → 상대방에게 "OO님이 당신을 소개하고 싶어해요. 수락할래요?" 알림 → 수락 시 프로필 공개
- "소개받고 싶어요" 토글은 이 opt-in의 상위 개념 — 토글 OFF면 소개 요청 자체가 안 감

## 5. 보상(썬구리) 지급 시점

**결정할 것:** 어느 액션에서 썬구리를 얼마나 줄지.

| 레퍼런스 | 보상 시점 | 보상 내용 | 효과 |
|---|---|---|---|
| Dropbox | 초대받은 사람이 **가입 완료** 시 | 양쪽 각 500MB | 15개월 3,900% 성장. CAC 광고 대비 60% 저렴 |
| Uber (인도) | 초대받은 사람이 **첫 탑승 완료** 시 | 양쪽 각 $10-30 크레딧 | ROI 12배 |
| PayPal | **가입 시** 즉시 | $20 (초기) → $10 → $5 점진 축소 | 1,000만 유저 달성 후 보상 축소 |

**판단 근거:**
- 너무 이른 보상(가입만 해도) → 허수 유입 (PayPal이 $20→$5로 줄인 이유)
- 너무 늦은 보상(매칭 성사 시만) → 보상 빈도 너무 낮아서 동기 부여 안 됨
- **행동 단계별 분산 지급**이 가장 효과적

**→ 스펙 방향:**

| 액션 | 썬구리 | 근거 |
|---|---|---|
| 소개 전송 | +5 | 행동 자체를 유도. 이게 없으면 첫 행동이 안 일어남 |
| 상대방 수락 | +10 | 질 좋은 소개를 유도 (아무나 보내면 수락 안 됨) |
| 대화 시작 | +20 | Hinge: 매칭 후 70%가 메시지 안 보냄 → 대화 시작이 진짜 가치 |
| 매칭 성사 | +50 + 뱃지 | 최종 보상. 빈도는 낮지만 성취감 극대화 |

- 초기에는 넉넉하게 주고, 유저 풀 확보 후 점진 축소 (PayPal 패턴)

## 6. 카카오톡 공유 카드: 미리보기 깊이

**결정할 것:** 카카오로 공유했을 때 비유저에게 뭘 얼마나 보여줄지.

| 레퍼런스 | 미리보기 깊이 | 전환 장치 |
|---|---|---|
| Locket | 위젯에 사진 일부 노출 → 앱 열어야 전체 보임 | **2주 200만 가입**. "궁금해서 설치" 구조 |
| LinkedIn | 프로필 상단 3줄 + "더 보려면 로그인" | 검색 유입 → 가입의 핵심 퍼널 |
| Tinder Matchmaker | 공유 링크 → 앱 설치 유도 | 효과 수치 미공개 |

**판단 근거:**
- 다 보여주면 설치할 이유 없음
- 아무것도 안 보여주면 클릭 안 함
- **"궁금하게 만드는 적정 수준"**이 전환의 핵심

**→ 스펙 방향:**
- 카카오 공유 카드에 보여줄 것: **친구의 추천사 + 프로필 블러 처리 사진 + 기본 정보 (나이, 지역)**
- 안 보여줄 것: 상세 프로필, 선명한 사진
- CTA: "OO님이 소개해주고 싶은 사람이 있대요 → 내친소에서 확인하기"
- 핵심: 추천사가 훅. "얘 진짜 좋은 사람이야"가 보이면 궁금해서 누름

## 7. 일일 소개 제한

**결정할 것:** 하루에 소개를 몇 번까지 할 수 있게 할지.

| 앱 | 제한 | 효과 |
|---|---|---|
| Cerca | 하루 4명 | 희소성 → 선택에 진지함 부여. 85K 유저 3개월 |
| Coffee Meets Bagel | 하루 정오에 6명 제공 | 일 방문 습관 형성 → MAU 안정 |
| Thursday | 주 1일만 활성화 | 초기 화제 → 그러나 6일간 앱 사망 → **2025 실질 실패** |
| Hinge | 하루 8개 좋아요 (무료) | 무제한 스와이프 피로 방지 |

**판단 근거:**
- 무제한 = 스와이프 피로 + 소개 품질 하락
- 너무 적으면(Thursday 주1회) 습관 형성 실패
- 하루 3~5개가 "신중하게 고르되 매일 쓸 이유가 있는" 밸런스

**→ 스펙 방향:**
- MVP: **하루 3회 소개 가능**
- 근거: Cerca(4)와 CMB(6)의 중간값. 초기 유저 풀이 작으니 더 적게 시작
- 썬구리로 추가 소개권 구매 가능 (수익화 연결점)

## 8. 소개 결과 피드백: 주선자에게 뭘 알려줄지

**결정할 것:** 내가 소개한 두 사람의 진행 상황을 어디까지 공유할지.

| 정보 | 공개 여부 | 근거 |
|---|---|---|
| "수락했어요" | ✅ 공개 | 주선자 동기 부여의 첫 단계 |
| "대화 시작했어요" | ✅ 공개 | 썬구리 +20 트리거. "내 소개가 통했다" 성취감 |
| "매칭 성사!" | ✅ 공개 + 뱃지 | 최고 보상. 공유 카드 생성 트리거 |
| 대화 내용 | ❌ 비공개 | 프라이버시 침해 |
| 누가 거절했는지 | ❌ 비공개 | 관계 훼손 방지 |
| 거절 사실 자체 | ⚠️ 모호하게 | "아쉽게도 이번엔 연결이 안 됐어요" (누구 탓인지 안 밝힘) |

**판단 근거:**
- Wingman: 매칭 성사 알림이 주선자 리텐션의 핵심 동력
- Tinder Matchmaker: 피드백 없으면 → "내가 뭘 한 건지 모르겠음" → 이탈
- Hinge: 매칭 후 70%가 메시지 안 보냄 → "대화 시작" 알림이 주선자에게 가치 있음

**→ 스펙 방향:** 진행은 알려주되, 실패의 책임은 안 보이게.
