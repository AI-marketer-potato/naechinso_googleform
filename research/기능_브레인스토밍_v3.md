# 내친소 기능 설계 — 코어 중심 v3

> **핵심 목표: 주선/추천사 기능을 "사람들이 쓰는 코어 기능"으로 만들어서 CAC를 줄인다**
> 마지막 업데이트: 2026-03-26

---

## 설계 원칙

```
주선 행동이 쉽고 재밌다
  → 주선이 자주 일어난다
    → 소개받는 사람이 늘어난다
      → 소개받은 사람이 다시 주선자가 된다
        → 유저가 유저를 데려온다 = CAC ↓
```

**모든 기능은 이 루프의 어딘가를 강화해야 존재 이유가 있다.**
루프에 기여하지 않는 기능은 만들지 않는다.

---

## 루프 기여도 기준

| 기능이 강화하는 것 | 의미 | 실패하면 |
|-------------------|------|---------|
| **주선을 쉽게** | 소개 행동의 마찰을 줄인다 | 주선 자체가 안 일어남 |
| **주선을 믿게** | 소개의 품질/신뢰를 높인다 | 수락률이 떨어짐 |
| **주선을 계속하게** | 주선자가 반복하도록 동기를 준다 | Ship처럼 주선자 이탈 |
| **주선이 퍼지게** | 주선 행동이 자연스럽게 새 유저를 데려온다 | 유료 광고 의존 = CAC ↑ |

## 기능 목록

### 주선을 쉽게 — 소개 행동의 마찰 제거

#### 1. 피드 즉시 공유 "이 사람 어때?"

- **뭘 하는 기능**: 피드 카드 → "친구에게 소개" 버튼 → 친구 선택 → 전송. 3초
- **루프 기여**: 주선 행동의 진입 장벽을 0에 가깝게 낮춤. 이게 없으면 루프 자체가 안 돌아감
- **핵심 UX**: 메시지 없이도 전송 가능 (TikTok 방식). 프리셋 메시지 3~4개 + 자유입력

```
피드 카드 → [친구에게 소개] 버튼
→ 친구 목록 (최근 대화 순)
→ 프리셋: "너 스타일이야 ㅋㅋ" / "취향 저격 아니야?" / "진지할 것 같아서"
→ 전송 → 썬구리 +5
```

| 레퍼런스 | 설명 | 내친소 어레인지 |
|---------|------|---------------|
| **TikTok 공유** | 1탭으로 친구에게 DM 공유. 메시지 없어도 됨 | 동일한 즉시성 적용 |
| **Hinge Send Profile** | 매칭 전 프로필을 친구에게 보내서 의견 묻기 | "의견 묻기"가 아니라 "소개"로 프레이밍 |

**목업**: `mockups/all_mockups.html#feature-1`

#### 3. 양방향 동시 소개

- **뭘 하는 기능**: 내 싱글 친구 A와 B를 골라 동시에 소개. 양쪽 수락 시 연결
- **루프 기여**: 한국 소개팅 문화("아는 사람이 소개해주는 것")를 그대로 디지털화. 가장 자연스러운 주선 형태
- **#1과의 관계**: #1이 "카드→친구" 일방향이면, 이건 "친구↔친구" 양방향

```
태양 👤     ↔     지수 👤
  ↑                  ↑
  └── 주선자: 나 ──┘

→ 양쪽에게 동시 알림
→ 한 쪽 거절 시 상대에게 알리지 않음
```

| 레퍼런스 | 설명 | 내친소 어레인지 |
|---------|------|---------------|
| **Hinge Matchmaker** | 두 사람 선택 → 아이스브레이커 자동 전송 | 자동이 아닌 주선자의 한마디가 전달됨 |
| **LinkedIn Get Introduced** | 공통 지인 통해 소개 요청 | 주선자가 능동적으로 두 사람을 매칭 |

**목업**: `mockups/all_mockups.html#feature-3`

#### 13. 구체적 프로필 반응

- **뭘 하는 기능**: 소개할 때 프로필의 **특정 부분을 하이라이트**해서 보냄
- **루프 기여**: "왜 이 사람을 소개하는지" 이유가 구체적 → 수락률 상승 → 주선 성공률 ↑ → 주선자 동기 ↑
- **#1의 업그레이드**: 단순 전송 → 맥락이 있는 전송

```
"야 이 사람 3번째 여행 사진 봐, 너 취향이잖아"
→ 해당 사진이 하이라이트된 상태로 친구에게 전달
```

| **Hinge 프로필 반응** | 특정 사진/답변에 직접 반응 → 대화 시작률 ↑ | 주선자 맥락으로 전환: "내가 왜 이걸 보여주는지" |

### 주선을 믿게 — 소개의 품질과 신뢰

#### 4. 추천사 먼저 보기

- **뭘 하는 기능**: 소개받았을 때 프로필 사진이 아닌 **친구의 추천사가 먼저** 보이는 카드 구조
- **루프 기여**: 내친소의 "친구 보증" 철학을 시각적으로 보여주는 핵심 차별점. 수락률을 높여 루프 회전 속도를 올림
- **핵심 UX**: 카드 상단에 "민준이가 소개해줬어요" + 추천 한마디 → 스크롤하면 프로필

```
┌─────────────────────┐
│  👤 민준이가 소개    │
│  "걔 진짜 성격 좋아" │
│─────────────────────│
│  [프로필 보기 →]     │
│  볼게!  /  나중에    │
└─────────────────────┘
```

| 레퍼런스 | 설명 | 내친소 어레인지 |
|---------|------|---------------|
| **Cerca 프로필 순서** | 공통 친구 → 배경 → 사진 순. 외모 전에 신뢰 정보 노출 | "공통 친구" 대신 "추천사"를 최상단에 배치 |
| **S'More 블러 리빌** | 사진 블러 → 대화할수록 언블러 | "볼게!" 버튼 누르면 프로필 공개. 추천사가 첫인상 |

**목업**: `mockups/all_mockups.html#feature-4`

#### 6. 공통 친구 우선 노출

```
│ 민준 · 태양         │

└────────────────────┘
```

| **LinkedIn** | "공통 연결 N명" 뱃지 → 연결 수락률/메시지 응답률 ↑ | 피드 정렬에도 반영 |

#### 12. 추천사 스택 (누적 보증)

- **뭘 하는 기능**: 받은 추천사가 프로필에 누적. "친구 5명이 보증함" 뱃지. 많을수록 피드 우선 노출
- **루프 기여**: 추천사가 많은 사람 = 더 많이 노출 = 더 많이 소개받음 → 추천사 작성 동기 ↑ → 주선 루프 가속
- **#4와의 관계**: #4가 "추천사 한 개"를 보여주는 거라면, 이건 "누적"

```
프로필 하단:
━━━ 친구 5명이 보증 ━━━
🗣 "걔 진짜 성격 좋아" — 민준
🗣 "유머 감각 최고" — 현우
🗣 "진지하게 연애하고 싶어하는 애" — 도윤
[+2 더보기]
```

| 레퍼런스 | 설명 | 내친소 어레인지 |
|---------|------|---------------|
| **LinkedIn endorsements** | 스킬별 보증 횟수 누적 → 프로필 신뢰도 ↑ | "스킬" 대신 "성격/매력 포인트"에 대한 추천사 |

**목업**: `mockups/all_mockups.html#feature-12`

#### 16. "소개받고 싶어요" 토글

- **뭘 하는 기능**: 토글 — 켜면 친구가 나를 소개 가능, 끄면 불가
- **루프 기여**: 안전장치. 이게 없으면 Hinge처럼 "creepy" 반응 → 전체 루프 신뢰 붕괴
- **핵심 UX**: 기본값 OFF. opt-in 방식으로 안전감 확보

```
설정 > 소개 관리
[🔘 소개받고 싶어요]  ← ON
"친구들이 다른 사람에게 너를 소개할 수 있어요.
 소개 전에 항상 네 동의를 먼저 구해요."
```

| 레퍼런스 | 설명 | 내친소 어레인지 |
|---------|------|---------------|
| **Hinge Matchmaker** | 기본값 opt-out → "creepy" → 실패 | 기본값 OFF (opt-in) |
| **Thursday** | 의도적 참여 시점 선택 | 토글 = "나는 소개받을 준비됐어" 시그널 |

### 주선을 계속하게 — 주선자 리텐션

#### 15. 주선 결과 피드백 루프

- **뭘 하는 기능**: 소개 후 진행 상황을 주선자에게 알림
- **루프 기여**: **Ship이 망한 핵심 이유를 해결하는 기능.** 결과를 모르면 주선 동기 소멸 → 피드백이 행동 반복을 만든다
- **핵심 UX**: 대화 내용은 안 보이고 상태만 전달 (프라이버시)

```
[주선자에게 알림]
"너의 소개로 태양 ↔ 지수가 대화를 시작했어! 🎉"
→ 썬구리 +20

[매칭 성사 시]
"주선 성공! 🎊 태양이 고마워하고 있어"
→ 썬구리 +50 + 뱃지
```

| 레퍼런스 | 설명 | 내친소 어레인지 |
|---------|------|---------------|
| **Tinder Matchmaker** | 추천 후 결과 피드백 없음 → 반복 행동 없음 | 정면 보완: 단계별 상태 알림 + 보상 |
| **Ship 실패** | 주선자가 계속 써야 할 이유 없음 → 서비스 종료 | 피드백 + 보상으로 주선자 리텐션 해결 |

**목업**: `mockups/all_mockups.html#feature-15`

### 주선이 퍼지게 — 자연 확산으로 CAC ↓

> 이 기능들은 주선 행동 자체가 새 유저를 데려오는 구조를 만든다

#### 5. 카카오톡 소개 카드

- **뭘 하는 기능**: 내친소 카드를 카카오톡으로 공유. 비유저가 미리보기 → 앱 설치 → 소개 확인
- **루프 기여**: 주선 행동이 앱 밖으로 나가는 순간 = 바이럴. 소개를 확인하려면 앱이 필요 → 자연 설치

```
[카카오톡 공유 카드]
┌─────────────────────┐
│ 내친소 💌            │
│ "민준이가 너한테     │
│  소개해주고 싶은     │
│  사람이 있대!"       │
│ [내친소에서 확인 →]  │
└─────────────────────┘
```

| 레퍼런스 | 설명 | 내친소 어레인지 |
|---------|------|---------------|
| **Locket** | 앱 작동에 친구 설치 필수 → 앱스토어 1위 | "소개 확인하려면 앱 필요" = 동일 구조 |
| **Hinge 초대 링크** | 비유저에게 초대 링크 발송 → 설치 유도 | 카카오톡이라는 한국 맥락 적용 |

**목업**: `mockups/all_mockups.html#feature-5`

#### 14. 소개 = 입장권

- **뭘 하는 기능**: 신규 유저가 기존 유저의 소개로만 가입 가능. 월 3장 "소개 티켓"
- **루프 기여**: 가입 자체가 주선 행동 → 모든 신규 유저가 이미 주선 루프 안에서 들어옴. CAC = 0의 이상적 구조
- **주의**: 유저 풀이 충분해야 진입 장벽이 가치가 됨. 코어가 단단해진 뒤 도입 (Clubhouse 교훈)

| 레퍼런스 | 설명 | 내친소 어레인지 |

| **Raya** | 수락률 8%, 지인 추천 = 입장 코드 | 소개 티켓 3장/월 |
| **Clubhouse** | 초대장 2장 → FOMO → 하지만 핵심 가치 없어서 급락 | 코어 기능 먼저, 초대제는 나중에 |
