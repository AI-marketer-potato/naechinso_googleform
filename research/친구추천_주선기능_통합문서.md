## 1. 왜 이 기능인가 — 리서치 종합

### 내친소의 고유 자산

내친소에는 다른 데이팅앱이 갖기 어려운 자산이 두 가지 있다.
**"친구가 보증한다"는 신뢰 구조**와 **6:4 성비**. 이걸 레버리지로 쓰면
유료 광고 없이도 유저가 유저를 데려오는 루프를 만들 수 있다.

지금 주선 기능이 그 역할을 못 하는 이유는 단순하다.
주선자에게 아무 도구도, 아무 보상도 없기 때문이다.

> **핵심 질문:** 어떻게 하면 친구를 소개해주고 싶게 만들 수 있을까?

### 지금 내친소의 문제

| 증상 | 이유 |
|---|---|
| 주선 모드로 바꿔도 할 수 있는 게 없다 | 주선자용 액션이 설계되어 있지 않음 |
| 주선 모드와 소개받는 모드가 따로 논다 | 두 역할이 분리되어 있어 동기 약화 |
| 기존 데이팅앱과 다를 게 없다 | 알고리즘 매칭 + 추천사 구조는 동일 |

### 반드시 해야 할 것 — 역할 통합

**데이팅앱의 리텐션 패러독스** _(Amplitude 분석)_

데이팅앱은 구조적으로 이상한 앱이다.
매칭이 성공하면 두 명의 유저가 동시에 앱을 떠난다.
그래서 주선자를 별도 역할로 고정해두면, 그 사람에게 남는 이유가 없어진다.

**Ship의 실패 (Match Group, 2019~2022)**

> 🚢 **어떤 앱이었나**
> Match Group(Tinder 모회사)이 만든 "친구가 대신 소개팅해주는 앱". 2019년 출시.
> 앱 안에 crew(친구 그룹)를 만들고, crew가 당사자 대신 카드를 스와이프한다.
> 매칭이 성사되면 crew 전원에게 알림이 간다. "우리가 해냈어!" 공유 경험 설계.
> Bumble, Tinder 등 주요 앱과 연동해서 기존 프로필을 그대로 쓸 수 있었음.

- 전체 매칭의 53%가 crew의 대리 스와이프로 발생할 만큼 잘 작동했지만
- 2022년 서비스 종료 — 이유: **주선자(crew)가 계속 쓸 이유가 없었다**
- 당사자가 매칭되면 앱을 떠나고, crew는 남아서 다음 사람을 위해 또 스와이프해야 함 → 지속 동기 없음

```
지금                         목표
──────────────────────       ──────────────────────────────
[소개받는 모드]     →        하나의 피드에서
       +           →        내가 소개받기도 하고
[주선만 하기 모드]  →        내 친구를 소개해주기도 한다
```

**살아남은 앱은 이렇게 했다**

> 🛡️ **Wingman (2018~현재 생존)**
> "친구가 내 소개팅 프로필을 직접 관리해주는 앱." 영국 스타트업, 현재까지 운영 중.
> 핵심 구조: Single 모드(내 매칭 찾기) + Wingman 모드(친구 매칭 찾기)를 동시 운영.
> 친구가 프로필에 직접 추천사를 작성 — "얘의 좋은 점 / 나쁜 점 / 특이한 점" 세 항목.
> 주선을 많이 한 사람에게 리더보드 순위 + 뱃지 부여 → 주선자 리텐션 유지.
> Ship과 다른 점: **주선자 본인도 매칭 찾을 수 있어서** 앱을 떠날 이유가 없다.

> 💚 **Hinge Matchmaker (2017, 현재는 기능 통합)**
> "진지한 연애를 원하는 사람들의 데이팅앱" Hinge가 2017년 출시한 주선 기능.
> 내 연락처에서 Hinge를 쓰는 친구들을 확인하고, 두 사람에게 동시에 메시지 전송.
> "OO이(가) 너희 둘을 연결하고 싶어해" → 아이스브레이커로 대화 시작.
> 비유저 친구에게는 초대 링크 발송 → 신규 가입 유도 루프.
> 지금은 Hinge 앱 내에 흡수되어 "Suggest a profile" 형태로 통합 운영 중.

### 브레인스토밍 — 2 × 2 프레임

두 가지 질문으로 가능성을 나눠봤다.

- **어떤 앱에서 배울 수 있나?** 데이팅앱 vs 비데이팅앱
- **누가 공유의 주체인가?** 내가 친구를 소개 vs 친구가 나를 위해 소개

```
                    데이팅앱 관점       비데이팅앱 관점
내가 친구를 소개          A                   C
친구가 나를 소개          B                   D
```

**A. 데이팅앱 × 내가 친구를 소개**
- 피드에서 카드 발견 → "친구에게 소개" 버튼 탭 → 누구에게 보낼지 선택 → 추천사 한 줄 작성 → 상대방 수락 시 연결 + 썬구리 적립
- 레퍼런스: Ship(대리 스와이프), Wingman("친구의 좋은 점/나쁜 점/특이한 점" 작성)
- 주의: 친구 동의 없이 프로필 공유 시 신뢰 훼손 → 양쪽 opt-in 구조 필수

**B. 데이팅앱 × 친구가 나를 위해 소개**
- 친구가 피드에서 카드 발견 → 나에게 전달 → 피드에 "친구 추천" 태그로 별도 노출 → 내가 수락/거절
- 레퍼런스: Cerca(2025, $1.6M 시드, 6만 유저) — 연락처 기반 1·2촌 매칭, 공통 친구 표시
- 한국 소개팅 문화와 가장 정확히 맞닿아 있는 flow

**C. 비데이팅앱 × 내가 친구를 소개 (앱 밖 바이럴)**
- 인스타/틱톡 DM·카카오 단톡방으로 스크린샷 돌리는 행동이 이미 일상 → 이 행동을 내친소 안으로
- 레퍼런스: Locket — 가입 시 베스트 프렌드 10명 선택 강제 → 2주 만에 200만 가입, 광고비 ≈ 0

**D. 비데이팅앱 × 친구가 나를 위해 소개 (커뮤니티 맥락)**
- 레퍼런스: LinkedIn 상호 소개(양쪽 동의 구조), Thursday(90만 다운로드, 목요일 하루만 매칭)
- 내친소 적용: "오늘 주선 가능" ON/OFF 토글, 번개 모임/오프라인 이벤트 탭

### CAC를 낮추는 수치 근거

| 앱 | CAC | 마케팅 비용 |
|---|---|---|
| Bumble | ~$56 (IPO S-1 공시) | 2022년 $207.7M — 매출의 약 23% |
| Tinder | 비공개 | Match Group 매출 $1.94B (2024) |
| Hinge | 비공개 | 2024년 매출 $550M, YoY +38% |

Bumble은 2025년 유료 마케팅을 대폭 줄이고 **유기적 성장 전략으로 전환**을 선언했다.
리퍼럴로 들어온 유저는 광고로 들어온 유저보다 **획득 비용 50% 저렴**, **LTV 16% 높다**.

**K-factor: 바이럴의 강도를 측정하는 수식**

```
K = 사용자당 평균 초대 수 (i) × 초대→가입 전환율 (c)

K > 1.0   → 유료 광고 없이 자생적 성장 가능
K 0.3~0.7 → CAC 30~70% 절감 (여기만 해도 충분히 의미 있음)
```

| 사례 | K-factor | 결과 |
|---|---|---|
| Dropbox | ~0.7 | 15개월 만에 10만 → 400만 유저 (+3,900%) |
| Locket | 추정 >1.0 | 2주 200만 가입, 광고비 ≈ 0 |
| Uber 리퍼럴 | 비공개 | ROI 12배 |

매칭이 성공하면 두 명이 동시에 앱을 떠난다 → 전파자 풀이 줄어든다.
해결책: 매칭이 성사되어도 **주선자는 남아야 한다** → 역할 통합이 단순한 UX 문제가 아닌 이유.

**보상 설계 원칙 (Dropbox·Uber에서 검증된 것)**
1. **양면 보상** — 주선자와 소개받는 사람 모두 혜택
2. **앱 가치와 연결된 보상** — 현금보다 썬구리·프로필 열기 같은 핵심 기능
3. **이미 하던 행동을 공식화** — 카카오톡으로 하던 소개를 내친소 버튼 하나로

### 한국 시장이 특이한 이유

| 앱 | 규모 | 특징 |
|---|---|---|
| 위피 | 누적 663만, 설치자 33만 | **2023년 한국 데이팅앱 매출 1위** |
| 틴더 | 설치자 42만 | 글로벌 인지도 |
| 아만다 | 누적 500만+ | 기존 유저 심사 통과해야 가입 |

위피는 **데이팅앱이 아닌 소셜앱으로 포지셔닝**했다. "동네에서 놀 사람 찾기", "지역 이벤트" 기능으로 데이팅에 거부감 있는 유저도 유입 가능하게 만들었다.
한국 오프라인 소개팅 문화를 **대체**하면 저항이 생기고, **디지털화**하면 자연스럽게 흡수된다.

내친소의 6:4 여성 우위 성비는 지켜야 할 핵심 자산이다 — 여성 리텐션이 올라가면 → 남성 매칭률이 올라가고 → 남성 리텐션도 올라간다.

### 우선순위 정리

| 순위 | 방향 | 왜 | 기대 효과 |
|---|---|---|---|
| 🔴 필수 | 주선 + 소개받기 역할 통합 | Ship이 실패한 이유, Wingman이 살아남은 이유 | 주선자 리텐션 확보 → LTV 증가 |
| 🟠 높음 | 소개 성사 시 양면 썬구리 보상 | Dropbox 3,900%, Uber 12x ROI | K-factor 0.3~0.7 → CAC 30~70% 절감 |
| 🟡 중간 | 카카오 공유로 프로필 미리보기 | Locket "친구가 있어야 작동" 구조 | 앱 밖 바이럴 → 신규 유입 루프 |
| 🟢 고려 | 공통 친구 있는 카드 상단 노출 | Cerca 소셜 그래프 매칭 방식 | 매칭 품질 향상 → 리텐션 개선 |
| 🔵 탐구 | "오늘 주선 가능" 시간 희소성 토글 | Thursday 목요일 한정 매칭 | 주선자 참여율 상승 |

## 2. 글로벌 레퍼런스

> 목적: 데이팅앱 소셜/친구 기능, 비데이팅앱 바이럴 리퍼럴, B2B 소개 기능의 PO 인사이트 정리

### 2-1. 데이팅앱 소셜/친구 기능 사례

#### Ship (Match Group × Betches Media, 2019~2022)

- Match Group과 Betches가 공동 개발한 크루 기반 매칭 앱
- **Crew 시스템**: 친구를 "크루"에 초대 → 크루가 당사자를 위해 스와이프 대리 수행
- **결과**: 전체 매칭의 53%가 크루의 대리 스와이프에서 발생 — 기술적으로는 작동

| 실패 원인 | 설명 |
|---|---|
| 주선자 리텐션 제로 | 친구 매칭이 성공하면 당사자는 떠나지만, 주선자가 계속 써야 하는 이유 없음 |
| 역할 분리 문제 | 이미 연애 중인 사람이 쓸 이유 없음 |
| 보상 부재 | 성사시켜도 주선자에게 돌아오는 인센티브 없음 |

**PO 교훈**: 친구 소셜 기능이 기술적으로 잘 작동해도, **주선자가 계속 써야 할 이유**가 없으면 앱이 유지되지 않는다.

---

#### Wingman (2018~현재 운영 중)

- 친구가 대신 프로필을 작성하고 매칭하는 앱
- **Dual Mode**: Wingee(소개받는 사람)와 Wingman(주선자) 역할을 하나의 앱 안에서 자유 전환
- 프로필을 **친구가 직접 작성** ("good/bad/ugly" 항목으로 솔직하게) → 신뢰도 높음
- **복수 Wingman 허용** → 여러 친구가 동시에 주선 경쟁 → 리더보드 경쟁 생성

| 구분 | Ship | Wingman |
|---|---|---|
| 역할 구조 | 당사자 + 크루 분리 | 하나의 앱에서 역할 전환 |
| 주선자 보상 | 없음 | 리더보드 포인트 |
| 지속 생존 | 2022년 종료 | 현재까지 운영 |

**PO 교훈**: Wingman이 Ship보다 오래 살아남은 이유는 **역할 통합과 주선자 리워드**.

---

#### Hinge Matchmaker (2017)

- "연애 중인 사람도 싱글 친구를 위해 데이팅앱에 참여할 수 있다"는 컨셉
- Facebook 소셜 그래프 활용 → 두 사람 지정 → 양쪽에 동시 아이스브레이커 전송 → 연결 성사
- 초대 링크 → 비사용자에게 발송 → 신규 유저 유입 루프 생성
- 한계: "Hinge Matchmaker Is A Little Creepy" (BuzzFeed) — **동의 없는 프로필 공유는 신뢰 파괴**

**PO 교훈**: 아이스브레이커 자동 발송이 핵심 마찰(어떻게 말 걸지)을 제거했다. 동의 없는 소셜 그래프 노출은 실패 원인.

---

#### Cerca (2025 — Gen Z 소셜 그래프 데이팅)

- "date mutuals, not strangers" — Ex-Tinder/Match/OkCupid/Grindr 임원들이 투자, 시드 $1.6M
- 앱 설치 → 연락처 자동 동기화 → 매일 4명의 추천 (1촌 또는 2촌 연결 필수)
- 프로필 표시 순서: **공통 친구 먼저 → 배경 → 사진** (외모보다 연결 강조)
- 하루 4개 제한 → Tinder식 스와이프 피로 해소

**PO 교훈**: **프로필에서 사진보다 공통 친구를 먼저 보여주는 것** 자체가 신뢰 신호.

---

#### S'More (2019~현재)

- "Love Is Blind" 앱 버전 — 외모 블러 후 대화로 점진적 언블러
- 2021년 멤버십 2배 성장

### 2-2. 비데이팅앱 바이럴 리퍼럴 사례

#### Locket Widget (2021~현재)

- 친구 사진을 아이폰 홈 스크린 위젯에 실시간 표시 — 누적 8,000만+, DAU 900만+
- **"강제 동반 설치" 구조**: 위젯을 받으려면 상대방도 앱이 있어야 함 → 자연스럽게 설치 강제
- 유료 광고비 ≈ 0으로 출시 2주 만에 200만 가입, App Store 1위 달성

**PO 교훈**:
1. "앱이 작동하려면 친구를 초대해야 한다" 구조가 가장 강력한 바이럴 루프
2. 초대 문자 자동 생성 → 사용자가 직접 카피라이팅할 필요 없음 → 마찰 제거

#### BeReal (2020~2024 고성장, 2024년 Voodoo에 €5억 매각)

- 2022년 피크 7,350만 유저 → 2023년 DAU 600만으로 급락
- 캠퍼스 앰배서더 프로그램 + 알림 동시성으로 급성장했으나 핵심 가치 부재로 급락
- **PO 교훈**: 초기 바이럴은 설계 가능하지만, **지속 리텐션은 핵심 가치에 달려 있다**.

---

#### Clubhouse (2020~2021 급성장)

- 기존 유저에게 초대장 2장만 지급 → 초대장이 eBay·Craigslist에서 거래 → FOMO 극대화
- 2021년 2월 1,000만 유저 → 1개월 내 MAU 70% 이탈
- **PO 교훈**: 초대 희소성은 강력한 초기 성장 도구지만, 매일 돌아올 이유가 있어야 한다.

---

#### Discord (2015~현재)

- 서버 초대 링크 시스템 + "Discord 자체가 친구들과 있을 때만 가치 있는 앱" 구조
- **PO 교훈**: **앱 내 가치가 "혼자서는 경험 불가"인 구조**가 가장 강력한 초대 동기.

---

#### Duolingo Friend Streak (2023 출시)

- Friend Streak 1개 이상 있는 유저는 **일일 학습 완료율 22% 상승**
- 바이럴 공유 트리거: 연속 마일스톤(성취) + 황당한 문장(유머) + 리더보드 1~3위(경쟁)
- **PO 교훈**: 공유 트리거는 "성취 + 유머 + 경쟁" — 이 중 하나를 충족하는 순간을 설계해야 함.

---

#### Dropbox (2008~현재) — 리퍼럴 프로그램 교과서

| 항목 | 수치 |
|---|---|
| 성장율 | 15개월 만에 10만 → 400만 유저 (+3,900%) |
| K-factor | ~0.35 |
| 피크 기여율 | 일일 신규 가입의 35%가 리퍼럴에서 발생 |

- 추천자 + 추천받은 사람 모두 500MB 스토리지 지급 (양면 보상)
- **PO 교훈**: K-factor 0.35도 유료 광고 없이 39배 성장을 만들 수 있다. 내친소 **썬구리 양면 보상**의 근거.

---

#### Strava (2009~현재)

- Kudos(좋아요) + Segments + Leaderboard → 친구의 활동이 내 행동의 동기가 되는 구조
- 최초 10억 건 기록: 8년 소요 → 그 다음 10억 건: 6개월 소요 (소셜 기능 성숙 효과)
- **PO 교훈**: 리더보드 경쟁은 자연적인 공유 충동을 만든다.

---

#### Venmo (2009~현재)

- 거래 내역이 기본 공개 → 소셜 피드에 노출 → 비유저가 "내 친구들이 쓰는 앱" 인식 → 설치
- **PO 교훈**: 유저가 의도적으로 초대하지 않아도, **앱 안의 행동 자체가 외부에 노출**되는 구조. 단, **프라이버시 기본값은 항상 opt-in**으로 설계해야 함.

## 3. 브레인스토밍 — 액션 아이디어

> 목적: 기능 개발 전 아이디어 확장 및 레퍼런스 정리

### 전제: 두 가지 추천 맥락

| 구분 | 능동형 | 세렌디피티형 |
|---|---|---|
| 시작점 | 친구를 위해 상대를 찾으러 들어옴 | 내 피드 보다가 친구 생각남 |
| 동기 | 친구 중심 ("얘 좋은 사람 만났으면") | 상대 중심 ("이 사람 OO이랑 잘 맞겠다") |
| 추천사 무게 | 신중하고 정성스럽게 | 즉흥적, 한 줄이면 충분 |
| 소개받는 감정 | "나를 위해 일부러 찾아줬다" | "나 생각났대" 따뜻함 |
| 수락 심리 | 신뢰감 → 높은 기대 | 호기심 → 낮은 허들 |

---

### A. 주선자 관점 — 능동형 진입

**A-1. 친구 대신 피드 탐색** `역할통합` `능동형`

- 친구를 선택하면 → 그 친구 프로필 기준으로 필터링된 피드가 열림
- 마음에 드는 카드에 "소개하기" 탭
- 레퍼런스: Ship(53% 매칭이 크루 스와이프에서 발생), Wingman(Wingman 모드 별도 운영)
- 내친소: 친구 프로필의 "이런 사람 원해요"를 기반으로 피드 자동 필터링

---

**A-2. 정성 추천사 작성** `능동형`

- 소개하는 친구의 프로필에 추천사 추가: 성격 한 줄 / 추천하는 이유 / 어떤 사람 만나면 좋을지
- 레퍼런스: Wingman("좋은 점/나쁜 점/특이한 점" 3가지 항목), LinkedIn 추천서
- 내친소: 추천사는 소개받는 사람도 볼 수 있는 공개 콘텐츠. **진정성 있는 한 줄**이 핵심

---

**A-3. 양방향 동시 소개 (그룹 소개)** `역할통합` `능동형`

- 주선자가 A와 B 두 사람을 선택 → 둘 다 수락하면 연결
- 레퍼런스: Hinge Matchmaker(두 명에게 동시 아이스브레이커 전달), LinkedIn "소개 요청"
- 내친소: 한국 소개팅 문화에 가장 가까운 형태

---

### B. 주선자 관점 — 세렌디피티형 진입

**B-1. 피드에서 즉시 공유** `세렌디피티형`

- 피드의 모든 카드에 "친구에게 소개" 버튼 → 탭 → 친구 목록 → 선택 → 한 줄 메모(선택) → 전송
- 전체 흐름이 3초 이내에 완료되어야 함
- 레퍼런스: Hinge "Send Profile", 카카오톡 공유 행동을 앱 안으로

---

**B-2. 한 줄 메모 소개** `세렌디피티형`

- 태그/이모지/짧은 한 줄만으로도 전달 가능: `"너 스타일이야 ㅋㅋ"` / `#유머있음 #카페좋아함`
- 레퍼런스: Locket(텍스트 없이 이미지만 보내도 됨), Instagram DM 공유
- 내친소: 세렌디피티형은 **순간의 감정**이 동력 → 마찰이 1초라도 길면 그냥 넘김

---

**B-3. "오늘 생각난 사람" 루틴화** `세렌디피티형` `역할통합`

- 하루에 몇 명이나 주선자가 태그를 달았는지 요약 → 주선 행동을 **일상의 루틴**으로
- 레퍼런스: Thursday(목요일 하루 앱 활성화 → 희소성으로 참여율 상승)

---

### C. 소개받는 사람 관점

**C-1. "친구 추천" 전용 카드** `역할통합`

- 일반 피드와 분리된 "친구가 소개해준 사람" 섹션
- 카드에 "OO이가 소개해줬어요" + 추천사 한 줄 노출 → 알고리즘 추천과 시각적 구분
- 레퍼런스: Cerca(공통 친구 있는 카드를 피드 상단 우선 노출), Hinge "Your Turn" 섹션

---

**C-2. 수락 전 "추천사 먼저 보기"** `능동형` `세렌디피티형`

- 프로필 열기 전에 주선자의 추천사 텍스트가 먼저 보임
- 추천사가 없으면 → "OO이가 피드 보다가 너 생각이 났대요"
- 레퍼런스: Wingman(친구 소개글이 프로필 최상단), Netflix("OO님이 좋아할 것 같아요")
- 내친소: 추천사 내용보다 **"사람이 골랐다"는 사실** 자체가 신뢰의 근거

---

**C-3. 나중에 볼게 (북마크)** `역할통합`

- 소개 요청을 즉시 수락/거절하지 않고 저장 → "오늘 저녁에 볼게" 리마인더 설정 가능
- 레퍼런스: Hinge(즉각 반응 강제 안 함) — Bumble의 24시간 강제는 여성 피로도 높임
- 내친소: "지금 연애할 마음이 없어"도 앱을 안 지워도 되는 이유

---

### D. 주선자 동기 유지 — 보상과 게이미피케이션

**보상 지급 타이밍**

```
소개 요청 전송  →  썬구리 X점 (즉시 지급)
상대방 수락     →  썬구리 Y점
대화 시작       →  썬구리 Z점
매칭 성사       →  주선자 뱃지 + 썬구리 보너스
```

레퍼런스: Dropbox(전환 완료 시점에 양쪽 보상), Uber 리퍼럴 ROI 12배, Wingman 리더보드

내친소: 썬구리의 핵심 가치 = **프로필 열기** → 성공적인 소개를 많이 한 사람이 더 많은 프로필을 볼 수 있음

**D-1. 주선 고수 뱃지**
- 누적 소개 성사 수 기반 타이틀 → 프로필에 노출 → "주선 잘하는 친구"라는 정체성 형성
- 레퍼런스: Wingman 리더보드, Reddit 카르마

**D-2. 소개 성공률 통계**
- "OO님의 소개 성공률: 68%" / "지금까지 3커플 연결에 기여했어요"
- 레퍼런스: Hinge "We Met" 리포트, Duolingo 스트릭

---

### E. 앱 밖 바이럴 — CAC를 낮추는 액션

**E-1. 카카오 공유로 프로필 미리보기** `세렌디피티형`

```
주선자가 피드에서 카드 발견
→ "카카오로 공유" 선택
→ 친구에게 "이 사람 어때?" 메시지 전송
→ 링크 열면 → 프로필 일부 미리보기 (흐림 처리)
→ "전체 보기 위해 내친소 앱 설치" CTA
→ 설치 후 주선자와 자동 연결
```

레퍼런스: Locket(앱 없이는 사진을 볼 수 없음 → 설치 유도), Hinge Matchmaker 초대 링크

---

**E-2. "너 소개받기 기다리는 사람 있어" 초대** `비데이팅×친구가소개`

- 주선자가 비유저 친구를 소개하고 싶음 → 연락처로 "소개 대기 중" 상태 생성 → 카카오 메시지 발송
- "OO이가 너를 내친소에서 소개해주고 싶어해" → 설치 → 소개 카드 확인
- 레퍼런스: Hinge Matchmaker 초대, Facebook "친구가 태그했어요" — 이미 뭔가가 기다리고 있다는 알림이 설치 전환율 극대화

---

### F. 한국 문화 특화 액션

**F-1. 번개 소개 모임** `역할통합`
- 주선자가 소개한 커플을 오프라인 이벤트에 초대 → 주선자도 분위기 메이커로 참석
- 레퍼런스: Thursday(목요일 하루 앱 활성화 + 오프라인 이벤트 연동, 90만 다운로드)

**F-2. 단톡방 맥락 소개** `역할통합`
- 그룹 기능: 3~5명의 친구들이 하나의 "소개 그룹" 구성 → 그룹 안에서 "이 사람 어때?" 피드백 논의
- 레퍼런스: Ship crew 그룹, 카카오톡 단톡방 소개팅 (이미 한국에서 일반적인 행동 패턴)

---

### G. 추천사 = 프로필 자산 (누적되는 신뢰)

**G-1. 친구 추천사 프로필 노출**
- 친구들이 써준 한 줄들이 프로필 하단에 누적 노출 → "친구 3명이 이렇게 소개했어요" 섹션
- 레퍼런스: LinkedIn 추천서, Wingman(친구 소개글이 프로필 최상단), Airbnb 호스트 리뷰

**G-2. 추천사 요청하기**
- 내가 친구에게 "나에 대해 한 줄 써줄 수 있어?" 요청 가능
- 친구는 앱 설치 없이 링크만으로 짧게 작성 가능 (비유저 참여 허용)
- 레퍼런스: LinkedIn "추천 요청", Google 리뷰 요청 링크

---

### H. 네트워크 코드 연계 (입장 코드로서의 추천)

**H-1. 추천이 곧 입장 코드**
- 친구 추천 없이는 가입이 어렵거나 노출이 제한됨 → "아무나 들어오는 앱이 아니야" → 희소성 + 안전감
- 레퍼런스: Clubhouse(초대장 2장 지급 → eBay에서 거래됨), Discord(서버 초대 링크 없이 입장 불가)

**H-2. 취향 네트워크 코드**
- "갓생 모임", "러닝 진심인 사람들" 같은 취향 그룹 안에서 주선 발생
- 레퍼런스: Tapple(관심사 그룹 입장 → 그룹 안에서 매칭)

---

### I. 주선 = 유입 명분 (기능이 마케팅이 됨)

**I-1. 기능 개편이 광고 카피가 되는 구조**

```
지금: "주선하러 깐 거야" → 카피는 있지만 실제 기능이 약함 → 허언이 됨
목표: "주선하러 깐 거야" → 실제로 피드에서 친구한테 소개 보낼 수 있음 → 진짜가 됨
```

레퍼런스: Bumble BFF("친구 사귀러 왔어" → 데이팅앱 낙인 없이 가입), 위피(소셜앱 포지셔닝)

**I-2. 주선자 스토리 바이럴**
- 주선 성사 시 → "OO이가 두 사람을 연결했어요 🎉" 공유 가능한 카드 생성
- 레퍼런스: Duolingo 스트릭 공유(100일 달성 카드 → SNS 자동 공유), Strava 세그먼트 1위

---

### J. 그룹 주선 (커뮤니티 단위 참여)

**J-1. 소개 그룹 (3~5인)**
- 친한 친구들로 구성된 소개 그룹 → 그룹 안에서 "이 사람 어때?" 카드를 함께 논의 → 당사자 최종 결정
- 레퍼런스: Ship crew 구조(단, 그룹 리텐션 실패 → 보상 설계 필수)

**J-2. 취향 커뮤니티 내 주선**
- 취향 공동체 = 신뢰 공동체 → 같은 그룹 안의 소개는 더 높은 수락률
- 레퍼런스: Tapple(관심사 그룹 입장 후 매칭), 스카이피플(직장인 커뮤니티 → 데이팅은 자연 발생)

---

## 4. 추천 메시지 브레인스토밍

> 핵심 관점: "추천사"가 아니라 "친구한테 보내는 메시지"

### 전제: 프레이밍 전환

```
추천사 (formal)   →   친구한테 보내는 메시지 (casual)
리뷰를 작성한다   →   카카오로 친구한테 연락하는 것
```

핵심 질문: "이 사람 어때?" 라고 친구한테 카카오로 보내는 것과 내친소 안에서 소개하는 것이 같은 느낌이어야 한다.

---

### 1. 메시지 형태 — 어떤 방식으로 전달할 수 있나

#### 1-1. 이모지/반응 하나

- 설명 없이 이모지 하나만 보내는 것: 💘 / 👀 / 🔥 / 😂
- 레퍼런스: iMessage Tapback(1탭으로 반응), BeReal RealMoji(반응 사진 1장)
- PO 포인트: Facebook Reactions 기준 이모지가 있으면 공유율 33% 상승, 상호작용 57% 증가
- 적합한 맥락: 세렌디피티형 ("보다가 생각났어" — 설명이 필요 없는 순간)

#### 1-2. 태그 선택

- 미리 준비된 태그를 1~3개 탭해서 보내는 것
- 예시 태그: `#네스타일` `#유머있음` `#진지한만남원해` `#취향저격`
- PO 포인트: 태그 선택은 1초 이내 결정 가능 — 텍스트 입력 대비 이탈률 대폭 낮음
- 적합한 맥락: 능동형/세렌디피티형 모두

#### 1-3. 짧은 한 줄 텍스트

- 자유 입력이지만 글자 수 제한 (예: 최대 30자)
- 예시: "진짜 네 스타일인데ㅋㅋ" / "얘 생각났어" / "진지한 것 같아서"
- 레퍼런스: TikTok "send to friend"(메시지 입력 선택 사항), Instagram DM 공유
- PO 포인트: 글자 수 제한이 역설적으로 부담을 낮춤 — "길게 안 써도 된다"는 신호
- 적합한 맥락: 능동형 (친구를 위해 일부러 찾은 경우 — 한마디 더 하고 싶은 마음)

#### 1-4. 프리셋 문장 선택 (편집 가능)

- 미리 작성된 문장을 선택하면 내 이름으로 전송됨
- 예시 프리셋: "보다가 네 생각났어" / "이 사람 성격 좋을 것 같아" / "취향 저격 아니야?" / "진지하게 만나볼 만한 것 같아서"
- 레퍼런스: AngelList 소개 템플릿 자동 작성(응답률 ~50%), Spotify 공유 메시지
- PO 포인트: AngelList 연구 — 소개자(middleman)의 마찰을 줄이는 것이 전환율의 가장 큰 레버
- 적합한 맥락: 능동형 + 빠른 세렌디피티형

#### 1-5. 음성 메시지

- 15초 이내 음성 한 마디 녹음 후 전송
- 레퍼런스: WhatsApp 음성 메시지(문자보다 강한 사회적 유대 형성)
- PO 포인트: 68%가 2번 들어야 내용 파악 → 첫 소개보다 후속 대화에 더 적합
- 적합한 맥락: 능동형 (정성 들인 소개, 오랜 친구 사이)

---

### 2. 메시지를 보내는 타이밍 — 언제 자연스럽게 발생하나

#### 2-1. 피드 스크롤 중 즉시

- 카드를 보다가 → 공유 버튼 → 친구 선택 → 이모지 or 태그 → 3초 안에 완료
- 설계 원칙: 공유 버튼 탭 → 친구 선택 → 이모지 선택 → 전송까지 **화면 전환 0회**
- PO 포인트: 화면 이탈이 생기면 공유 행동 80% 이상 소실

#### 2-2. 친구 프로필 방문 중

- 앱 안에서 친구 프로필을 보다가 → "OO한테 어울릴 것 같은 사람 찾아줄까?" → 능동형 탐색 시작
- PO 포인트: 친구를 먼저 생각하는 맥락에서 진입 → 메시지 퀄리티가 올라감

#### 2-3. 매칭 성사 후 친구에게 알림

- 내가 소개해준 두 사람이 연결됨 → "OO와 XX가 연결됐어! 네 덕분이야" 알림
- 알림에서 바로 "잘됐다 🎉" 한 탭 반응 가능
- PO 포인트: 성사 순간이 가장 강력한 재공유 트리거 — 이 순간에 "친구 더 소개해주기" 유도

#### 2-4. 앱 밖에서 시작 (카카오 딥링크)

- 카카오에서 "이 사람 어때?" 를 받은 비유저 → 링크 클릭 → 설치 → 프로필 확인
- PO 포인트: "이미 뭔가가 기다리고 있어" 프레이밍이 단순 초대보다 설치 전환율 높음

---

### 3. 메시지를 받는 경험 — 어떻게 보여야 하나

#### 3-1. "친구 추천" 카드와 일반 카드의 시각적 분리

- 카드 상단에: "OO이가 보냈어 💌" + 이모지/태그/한 줄 메시지 먼저 보임 → 그 아래: 프로필 열기 버튼
- 레퍼런스: Tinder Matchmaker "friend recommended" 태그, Cerca 공통 친구 최상단 노출
- PO 포인트: 공통 지인 소개 → 응답률 70% 상승 (LinkedIn 연구)

#### 3-2. 메시지 먼저, 프로필 나중

- 순서: 친구 이름 + 메시지 → "프로필 열기" → 사진/정보
- 레퍼런스: S'More(사진 블러 처리, 대화가 먼저) — 여성 리텐션 개선 효과
- PO 포인트: 외모보다 맥락을 먼저 보는 설계 → 매칭 품질 상승, 여성 유저 리텐션에 긍정적

#### 3-3. 수락 전 "나중에 볼게" 옵션

- 즉시 수락/거절 강제하지 않음 → "오늘 저녁에 볼게" 저장 → 리마인더 알림
- PO 포인트: "지금 연애할 마음이 없어도 앱을 지우지 않는 이유" 설계

#### 3-4. 거절해도 친구한테 안 알려짐

- 소개받은 카드를 거절해도 소개해준 친구에게 알림 없음
- 소개해준 사람은 "수락됨" 알림만 받음 (거절은 미통보)
- PO 포인트: 거절 비용이 낮아야 수락 비용도 낮아짐. 양쪽의 심리적 안전망

### 4. 프레이밍 — 어떤 단어를 써야 하나

| 단어 | 심리적 무게 | 적합한 맥락 |
|---|---|---|

### 5. 보상 — 메시지를 보낼 동기를 어떻게 만드나

- 친구 추천 카드를 수락하면 썬구리 (능동적 참여 유도)
- "친구 추천으로 연결된 사람과의 대화" 우선 노출

## 5. 캐주얼 추천 메시지 UX 리서치

> "친구한테 문자 보내는 느낌"을 만드는 UX 패턴

### 핵심 발견 3가지

1. **마찰이 낮을수록 전달율이 높다.** 공유 행동은 입력 필드가 아니라 버튼 하나로 시작해야 한다. 미리 채워진 메시지(pre-filled)는 전혀 없는 것보다 전환율을 높이지만, 편집 가능해야 개인적으로 느껴진다.

2. **"보낸다(forward)"는 느낌과 "추천한다(recommend)"는 느낌은 심리적으로 다르다.** Forward는 정보 전달, recommend는 사회적 책임을 동반한다. 소개팅 맥락에서는 추천자의 책임감을 낮춰야 참여율이 올라간다 — "내가 보증한다"가 아니라 "혹시 어떨까 해서 보내봤어" 프레임이 필요하다.

3. **텍스트 입력 없이도 충분히 의미 있는 신호를 보낼 수 있다.** 반응(reaction), 이모지, 원탭 신호는 "보내도 될까?" 심리적 허들을 없애며, 메시지 없는 공유도 공유다.

---

### 1. 소셜앱의 캐주얼 공유 메커니즘

#### TikTok: "Shared Feed"와 친구에게 보내기

- 단일 영상 친구에게 보내기: 공유 아이콘 탭 → 수신자 선택 → [선택] 메시지 추가 → 보내기
- **핵심 UX 결정: 메시지 없이도 즉시 전송 가능. 텍스트는 선택이지 필수가 아니다.**
- 내친소 적용: 프로필 카드 공유 시 메시지 텍스트는 선택(필수 아님). 수신자 선택 UI에서 최근 대화 목록 자동 노출

---

#### Instagram DM 공유

- 공유 버튼 → 수신자 선택 → 메시지 입력창이 이미 열려 있음 → 보내기
- "뭔가 말해도 되는 공간"이 준비되어 있지만 비워도 됨
- **교훈**: 공유 행동은 현재 화면에서 즉시 시작되어야 한다. 다른 탭이나 메뉴로 이동하는 순간 사용자는 행동을 포기한다.

---

#### Snapchat: 에피소드성이 만드는 낮은 심리적 부담

- 사라지는 메시지 → "틀린 말 하면 어쩌지"라는 부담 없음
- 정제되지 않은 콘텐츠가 기본 → "잘 만들어야 한다"는 압박 없음
- **소개팅 추천메시지에 적용**: 추천 메시지가 영구 기록으로 남으면 추천자는 더 신중해진다. 메시지의 가시성과 지속성을 어떻게 설계하느냐가 참여율에 직접 영향을 준다.

---

#### BeReal RealMoji: 사진으로 반응하는 저마찰 신호

- 친구의 BeReal에 이모지 탭 → 셀카 카메라 열림 → 즉석 표정 사진이 이모지 자리를 대체
- 텍스트 입력 없음, "좋아요"보다 더 개인적이지만 댓글보다 훨씬 쉬움
- **내친소 적용**: 주선자가 프로필 카드를 보낸 후, 수신자가 이모지나 짧은 셀카를 돌려보내는 구조 → 텍스트 없는 "관심 신호"

---

#### iMessage Tapback: 원탭으로 충분한 이유

- 메시지당 1개 제한 → 제약이 결정을 빠르게 만듦
- 타이핑 불필요 → 상대가 보낸 직후 즉시 처리 가능
- 소개팅 맥락: 친구에게 소개받은 프로필을 보고 "글쎄..."라고 느낄 때 거절 문자 쓰기 싫은 상황 — 원탭 반응이 해결한다.

---

#### KakaoTalk 스티커와 빠른 답장

- ACM Human-Computer Interaction 2024 연구: 스티커를 선택하는 이유는 "감정 정확도"보다 "내가 그 사람에게 어떻게 보이고 싶은가"가 더 중요 — 스티커는 감정 표현이자 관계 표현이다.

---

### 2. 음악/엔터테인먼트 추천: 캐주얼 메시지로서의 공유

#### Spotify Messages (2025년 8월 출시)

- 출시 후 4,000만 명이 3억 4,000만 건의 메시지 발송
- 음악 공유가 "들어봐"라는 텍스트 없이도 "내가 지금 이걸 듣고 있어"라는 존재감 공유로 작동
- **내친소 적용**: "지금 이 프로필 보고 있어"라는 실시간 신호를 친구에게 보낼 수 있는 구조

---

#### Letterboxd: 왜 친구 리뷰가 형식적이지 않은가

- 별점 중심 / "봤어요" 체크만 해도 됨 / 리뷰 길이 제한 없음 (한 줄도 OK, 에세이도 OK)
- **내친소에 적용**: 추천사가 "잘 작성된 소개 글"이어야 한다는 암묵적 기대를 없애야 한다. 한 단어, 이모지, 짧은 농담도 유효한 추천으로 취급되는 문화 설계가 필요하다.

---

### 3. 친구 추천이 행동을 이끄는 비데이팅 앱들

#### Google Maps: 장소 공유에 노트 추가

- **배울 점**: 추천 메시지는 "편지"가 아니라 "라벨"처럼 느껴져야 한다.

---

#### Airbnb Wishlist 공유: 발견된 실패 패턴

- 실제 사용자 중 Wishlist 공유 기능을 아는 비율은 약 9%에 불과 — 공유하라는 진입점이 기능 안에 묻혀 있음
- **교훈**: 공유 기능이 존재한다고 사용되는 것이 아니다. "이 카드 괜찮은데 OO한테 어울릴 것 같다"는 생각이 드는 순간, 그 생각이 사라지기 전에 공유 버튼이 보여야 한다.

---

### 4. 데이팅 인접 앱의 캐주얼 사회적 증거

#### Tinder Matchmaker: 친구가 매칭을 대신 고르는 기능

- 친구가 Tinder 계정 없이도 게스트로 참여 가능 — 초대 장벽 낮음
- 최종 결정권은 본인 → 추천자 부담 줄어듦
- 아쉬운 것: 4시간 세션 제한이 너무 짧음 / 친구가 추천 이유를 남길 수 없음 / 추천자가 결과를 알 수 없음 → 피드백 루프 없음

---

#### Hinge: 저마찰 긍정 신호의 설계

- "당신 좋아요"가 아니라 "이 사진 웃겼어요" → 더 구체적이고 덜 무서운 신호
- **소개팅 추천에 적용**: "이 사람 좋을 것 같아"보다 "이 사람 사진 3번째 거 너 취향이잖아"가 더 자연스럽다.

---

#### Coffee Meets Bagel: 큐레이션 프레이밍

- 하루 정오에 소수의 추천 매칭 제공 → 한 명 한 명이 더 진지하게 보임
- **친구 추천에 적용**: "오늘 내 친구 추천" 프레이밍으로 하루에 1~3명만 소개 가능하게 제한하면 오히려 더 가치 있는 신호로 인식된다.

---

### 5. 캐주얼 vs 공식 추천의 심리학

#### 왜 친구 추천이 공식 리뷰보다 신뢰받는가

- 소비자의 88%는 아는 사람의 추천을 모든 형태의 마케팅보다 더 신뢰한다 (Buyapowa, 2025)
- 친구/가족 추천 vs 브랜드 메시지: 신뢰도 5배 차이 (LinkedIn Research)

**심리적 메커니즘 3가지**
1. **이해충돌 없음**: 친구는 나를 팔려고 추천하는 게 아니다.
2. **취향 유사성**: 나와 비슷한 사람이 좋아하면 나도 좋아할 가능성이 높다.
3. **감정적 투자**: 친구가 추천했다는 것은 나를 위해 시간을 썼다는 의미다.

---

#### "추천사"를 "공식 추천서"처럼 느끼게 만드는 요소들 — 피해야 할 것들

| 요소 | 공식적으로 느껴지는 이유 | 대안 |
|---|---|---|
| 긴 입력창 | 긴 글을 써야 한다는 압박 | 짧은 입력창 또는 캐릭터 제한 표시 |
| "추천사 작성" 레이블 | 공문서 느낌 | "한마디 남기기", "어떤 점이 떠올랐어?" |
| 필수 입력 | 쓰지 않으면 진행 불가 | 선택 사항으로 설계 |
| 제출 후 수정 불가 | 신중하게 써야 한다는 인식 | 수정 가능 |
| 상대방에게 즉시 공개 | 부담 증가 | "먼저 나한테만 보이고, 수락 시 공개" |
| 격식체 요청 | 공식적인 톤 강제 | 구어체 예시 프롬프트 |

---

#### 짧은 신호 vs 긴 추천의 전환율

| 신호 유형 | 발신자 부담 | 수신자 신뢰도 | 전환율 추정 |
|---|---|---|---|
| 이모지 반응만 | 매우 낮음 | 낮음 | 높음 (보내는 사람 많음) |
| 원탭 + 미리채워진 메시지 | 낮음 | 중간 | 높음 |
| 2~3줄 자유 텍스트 | 중간 | 높음 | 중간 |
| 5줄 이상 자유 텍스트 | 높음 | 매우 높음 | 낮음 (쓰는 사람 적음) |
| 음성 메시지 | 높음 | 매우 높음 | 매우 낮음 |

**최적 설계**: 낮은 진입 장벽(이모지 또는 원탭) + 선택적 텍스트 추가 옵션.

---

### 6. UX 패턴 종합: 공유를 자연스럽게 만드는 결정들

#### 미리 채워진 메시지 (Pre-filled templates)

- 중요한 조건: **편집 가능해야 개인적으로 느껴진다.**
- 나쁜 예: "OOO님이 내친소에서 당신을 만나보라고 추천했습니다."
- 좋은 예 (편집 가능): "야 이 사람 너 취향 아니야? 한번 봐봐"

---

#### 이모지/반응 전용 옵션

- Facebook Reactions 출시 이후 첫 1년 내 3,000억 건의 반응 (하루 8억 건)
- 이모지를 포함한 게시물은 포함하지 않은 것보다 공유율 33%, 인터랙션 57% 높음
- **설계 원칙**: 반응 옵션을 공유 플로우의 첫 단계로 배치.

---

#### "포워딩"이 아닌 "소개하기" 프레이밍

| 표현 | 심리적 무게 | 참여율 예상 |
|---|---|---|
| "추천서 작성하기" | 무거움 | 낮음 |
| "친구에게 추천하기" | 중간 | 중간 |
| "소개해주기" | 가벼움 | 높음 |
| "한번 보내보기" | 매우 가벼움 | 높음 |
| "이 사람 어때?" | 최소 부담 | 가장 높음 |

---

#### 공유 타이밍: 생각이 나는 순간에 잡아라

- Airbnb Wishlist 실패(9% 인지율): 공유 진입점이 기능 안에 묻혀 있었기 때문
- Tinder Matchmaker의 교훈: 세션 링크를 만들어서 친구에게 보내는 방식은 맥락 전환을 강제한다. **"지금 보는 이 카드를, 지금 바로" 보낼 수 있는 구조가 훨씬 자연스럽다.**

---

#### 양방향 수락 구조 (LinkedIn 방식)

```
A (주선자) → "이 사람 어때?" + 프로필 일부 → B (소개받을 친구)
B → "볼게" / "괜찮아"
B가 수락 → 프로필 전체 공개
B가 수락 + 좋아요 → 상대방(C)에게 "OO님의 친구가 관심 있어요" 알림
C → 수락/거절
양쪽 수락 → 연결
```

**왜 이것이 중요한가**
- 소개자 보호: 동의 없이 프로필을 공유당한 사람이 불쾌감을 느끼면 내친소 신뢰 훼손
- 수신자 자율성: 친구의 추천이지만 최종 선택은 본인

---

### 7. 내친소 추천 메시지 설계 권고안

**핵심 원칙 5가지**

**원칙 1: "추천서"가 아닌 "문자" 톤**
입력창 레이블, 플레이스홀더 텍스트, 버튼 레이블 모두 구어체. "추천 내용을 입력하세요"가 아니라 "어떤 점이 생각났어? (선택)"

**원칙 2: 없어도 괜찮다**
메시지 없이 프로필만 보내도 유효한 행동. 이모지 하나도 유효한 신호. 긴 추천사는 쓰고 싶을 때만.

**원칙 3: 카드를 보는 순간이 공유 타이밍**
피드 카드에 공유 버튼이 바로 보여야 한다. 다른 화면으로 이동하면 생각이 사라진다.

**원칙 4: 발신자와 수신자 모두의 부담을 낮춰라**
발신자: "틀린 추천을 하면 어쩌지" 부담 → "소개만 했을 뿐, 결정은 네가 해"
수신자: "이걸 어떻게 반응해야 하지" 부담 → 원탭 반응 옵션 제공

**원칙 5: 짧고 구체적인 추천이 길고 막연한 것보다 낫다**
Hinge의 접근처럼 — "이 사람 좋을 것 같아"보다 "이 사진 너 취향이잖아"

**최소 viable 플로우 (MVP)**

```
피드 카드 보는 중
→ "친구에게 소개하기" 버튼 (카드 하단, 작게)
→ 친구 선택 (최근 대화 목록, 또는 연락처)
→ [선택] 빠른 반응: 이모지 3개 (예: 😏 👀 🔥) 또는 짧은 텍스트
→ 보내기
→ 수신자에게 "OO님이 이 사람 어떤지 물어봐" 알림 (카드 일부 미리보기 포함)
→ 수신자: 볼게 / 됐어 탭
→ "볼게" 선택 시 프로필 전체 공개
```
