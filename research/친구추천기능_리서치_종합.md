# 내친소 "친구 주선 기능" — 레퍼런스 리서치 종합

## 핵심 배경
내친소의 핵심 자산: **친구 추천 기반 신뢰 시스템** + **6:4 성비**
목표: 위피/당근 수준의 플랫폼으로 성장, 주선 기능을 레버리지로 활용

## 현재 내친소 UI 분석 (앱 실제 스크린샷 기반)

### "소개 받기" 모드 (기본)
- 하단 탭: 소개 받기 아이콘(🟠) / 쪽지 / 마이페이지
- "주선만 하기 모드로 전환이 완료됐어!" 안내 화면
- 상대방 프로필 사진 + "프로필 열기" 버튼
- 거절 외에는 할 게 없음

### 마이페이지
- 상단 배너: "친구 초대 시 무제한 100 씬구점" 초대하기
- 두 가지 탭: "소개 받을래" / "주선만 할래"
- 프로필 + 내친소 Pro / 미리보기·수정하기
- 하단 버튼: "내 추천사 받기" / "친구 초대하기"
- "주선만 하기 모드로 전환할래?" 안내

### 핵심 문제점
- "주선만 하기" 모드로 전환해도 거절 외에 할 수 있는 게 아무것도 없음
- 주선자한테 주어지는 도구나 액션이 전혀 없는 상태
- 기존 데이팅앱과 차별점이 없음 (알고리즘 매칭 + 선택적 추천사)

## 레퍼런스 리서치

### 관점 1. 데이팅앱 관점

**A. 대리 스와이프형**
- Ship (Match Group, 2019) — crew 그룹으로 대리 스와이프, 빠른 성장 후 2022 종료 (주선자 리텐션 실패)
- Wingman (2018~현재) — 대리 스와이프 + 리더보드 게이미피케이션으로 생존

**B. 소셜 그래프 매칭형**
- Cerca (2025, $1.6M 시드) — 연락처 기반 1촌/2촌 매칭, 공통 친구 먼저 표시, 6만 유저
- Hinge Matchmaker (2017) — 비유저 친구 초대 메커니즘

### 관점 2. 커뮤니티(비데이팅) 관점

**C. 친구 초대 기반 바이럴 루프형**
- Locket — "친구가 있어야 앱 작동" 구조, 2주 만에 200만 가입, DAU 900만+

**D. 커뮤니티→데이팅 전환형**
- 블라인드→블릿 — 커뮤니티 먼저, 데이팅은 자연 발생
- 스카이피플 — 인증 소개팅 + 직장인 커뮤니티, 55만 유저

**E. 오프라인 이벤트 연동형**
- Thursday — 목요일 하루만 매칭 + 오프라인 이벤트, 90만 다운로드
- Bumble BFF — Geneva 인수, 그룹/커뮤니티 기반 전환

## MECE 프레임워크

### 축 1: 주선자의 역할 범위
프로필만 추천 (Cerca) → 직접 매칭 행위 (Ship/Wingman) → 프로필 대리 작성 (Wingman 추천서 강화)

### 축 2: 주선 대상의 앱 사용 여부
이미 유저인 친구 주선 (리텐션) → 비유저 친구 초대해서 주선 (acquisition)

### 축 3: 참여 동기 설계
이타적 동기 → 게이미피케이션 (리더보드, 뱃지) → 커뮤니티 소속감 → 직접적 보상 (양면 인센티브)
