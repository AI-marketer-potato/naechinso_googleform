# 내친소 친구 주선 기능 — 리서치 종합

> 마지막 업데이트: 2026-03-25

---

## 왜 이 기능인가

내친소에는 다른 데이팅앱이 갖기 어려운 자산이 두 가지 있다.
**"친구가 보증한다"는 신뢰 구조**와 **6:4 성비**. 이걸 레버리지로 쓰면
유료 광고 없이도 유저가 유저를 데려오는 루프를 만들 수 있다.

지금 주선 기능이 그 역할을 못 하는 이유는 단순하다.
주선자에게 아무 도구도, 아무 보상도 없기 때문이다.

> **핵심 질문:** 어떻게 하면 친구를 소개해주고 싶게 만들 수 있을까?

---

## 지금 내친소의 문제

| 증상 | 이유 |
|---|---|
| 주선 모드로 바꿔도 할 수 있는 게 없다 | 주선자용 액션이 설계되어 있지 않음 |
| 주선 모드와 소개받는 모드가 따로 논다 | 두 역할이 분리되어 있어 동기 약화 |
| 기존 데이팅앱과 다를 게 없다 | 알고리즘 매칭 + 추천사 구조는 동일 |

---

## 반드시 해야 할 것 — 역할 통합

### 왜 분리하면 망하나

**데이팅앱의 리텐션 패러독스** _(Amplitude 분석)_

데이팅앱은 구조적으로 이상한 앱이다.
매칭이 성공하면 두 명의 유저가 동시에 앱을 떠난다.
그래서 주선자를 별도 역할로 고정해두면, 그 사람에게 남는 이유가 없어진다.

**Ship의 실패 (Match Group, 2019~2022)**

- 친구 그룹(crew)이 당사자 대신 스와이프하는 기능
- 전체 매칭의 53%가 crew의 대리 스와이프로 발생할 만큼 잘 작동했지만
- 2022년 서비스 종료 — 이유: **주선자가 계속 쓸 이유가 없었다**

### 어떻게 통합하나

```
지금                         목표
──────────────────────       ──────────────────────────────
[소개받는 모드]     →        하나의 피드에서
       +           →        내가 소개받기도 하고
[주선만 하기 모드]  →        내 친구를 소개해주기도 한다
```

**살아남은 앱은 이렇게 했다**

- **Wingman (2018~현재 생존)** — 내 매칭도 찾고, 친구 매칭도 찾는 구조를 한 앱 안에 넣었다. 리더보드로 "누가 가장 많이 성사시켰나" 경쟁을 붙여 주선자 리텐션 유지.
- **Hinge Matchmaker (2017)** — 친구 중 Hinge 유저를 확인하고 두 사람에게 동시에 아이스브레이커 메시지를 보내는 구조. 초대 링크 → 신규 유저 유입 루프까지 연결.

## 브레인스토밍 — 2 × 2 프레임

두 가지 질문으로 가능성을 나눠봤다.

- **어떤 앱에서 배울 수 있나?** 데이팅앱 vs 비데이팅앱
- **누가 공유의 주체인가?** 내가 친구를 소개 vs 친구가 나를 위해 소개

```
                    데이팅앱 관점       비데이팅앱 관점
내가 친구를 소개          A                   C
친구가 나를 소개          B                   D
```

---

### A. 데이팅앱 × 내가 친구를 소개

> "피드를 보다가 '이 사람 우리 친구랑 잘 맞겠다'는 생각이 드는 순간을 앱 안에 담는다"

**어떻게 흘러가나**
```
내가 피드에서 카드 발견
→ "친구에게 소개" 버튼 탭
→ 누구에게 보낼지 선택
→ 추천사 한 줄 작성
→ 상대방이 수락하면 연결, 나에게 씬구점 적립
```

**레퍼런스**
- Ship: 대리 스와이프, 매칭 성사 시 crew에게 알림
- Wingman: "친구의 좋은 점 / 나쁜 점 / 특이한 점"을 직접 작성해서 프로필에 추가

**내친소에서 특별한 이유**
주선자가 직접 쓴 추천사 → 내친소의 신뢰 시스템과 자연스럽게 연결된다.

**주의할 점**
- 친구 동의 없이 프로필이 공유되면 신뢰 훼손. LinkedIn처럼 양쪽 opt-in 구조 필수.
- Ship이 망한 이유 = 보상 부재. 씬구점이든 뱃지든 명확한 인센티브가 있어야 한다.

---

### B. 데이팅앱 × 친구가 나를 위해 소개

> "알고리즘이 고른 사람보다, 나를 아는 친구가 고른 사람이 더 믿음직하다"

**어떻게 흘러가나**
```
친구가 내 취향을 알고 있음
→ 피드에서 카드 보다가 "이거 OO한테 어울리겠다" 생각
→ 나에게 전달
→ 피드에 "친구 추천" 태그로 별도 노출
→ 내가 수락/거절
```

**레퍼런스**
- **Cerca (2025, $1.6M 시드, 6만 유저)**: 연락처 기반 1·2촌 매칭, 공통 친구 표시 → 앱 안에서 지인 네트워크가 보임
- **Hinge "Send Profile"**: 특정 프로필을 친구에게 DM으로 전송, "이 사람 어때?" 한마디

**내친소에서 특별한 이유**
한국 소개팅 문화와 가장 정확히 맞닿아 있는 flow다.
"앱이 골라준 사람"이 아니라 "친구가 골라준 사람" — 이게 내친소가 다른 앱과 달라야 하는 이유.

---

### C. 비데이팅앱 × 내가 친구를 소개 (앱 밖 바이럴)

> "사람들은 이미 카카오톡으로 소개팅을 하고 있다. 도구만 없을 뿐."

**현상**
- 인스타/틱톡 DM으로 "이 사람 소개해줘" 는 이미 일상
- 카카오톡 단톡방에서 스크린샷 돌리는 소개팅도 흔함
- 문제는 도구가 없어서 불편하다는 것 — 이 행동을 내친소 안으로 가져오면 된다

**Locket의 "친구가 있어야 작동하는" 구조**
- 가입 시 베스트 프렌드 10명 선택 강제 → 앱이 작동하려면 친구를 초대해야 함
- 결과: **2주 만에 200만 가입**, 유료 광고비 ≈ 0, 누적 9,000만 다운로드

**내친소 적용 아이디어**
- 카카오 공유하기로 프로필 일부만 미리보기 → 앱 설치 유도
- "OO님이 당신을 내친소에서 만나봤으면 해요" 공유 메시지
- "친구가 1명 이상 있어야 소개 가능" 구조 → 가입과 동시에 초대 행동 발생

---

### D. 비데이팅앱 × 친구가 나를 위해 소개 (커뮤니티 맥락)

> "데이팅앱이라고 생각하지 않는 사람도 쓸 수 있게 되면, 유입 풀이 넓어진다"

**레퍼런스**

- **LinkedIn 상호 소개**: 2촌 대상에게 그룹 메시지로 두 사람 동시 소개. 핵심은 **양쪽 동의 구조** — 소개받기 전에 "받을래요?" 먼저 묻는다.
- **스카이피플 (55만 유저)**: 직장인 커뮤니티가 1차, 만남은 2차로 자연 발생
- **Thursday (90만 다운로드)**: 목요일 하루만 매칭 활성화 → 희소성으로 참여율 상승

**내친소 적용 아이디어**
- "오늘 주선 가능" ON/OFF 토글 → 주선자가 여유 있을 때만 참여
- 번개 모임 / 오프라인 이벤트 탭 → 커뮤니티로 진입, 만남은 자연 발생
- 단톡방 맥락 활용: "이 사람 어때?" 를 그룹 안에서 물어볼 수 있는 기능

## CAC를 낮추는 수치 근거

### 데이팅앱은 광고비가 얼마나 드나

| 앱 | CAC | 마케팅 비용 |
|---|---|---|
| Bumble | ~$56 (IPO S-1 공시) | 2022년 $207.7M — 매출의 약 23% |
| Tinder | 비공개 | Match Group 매출 $1.94B (2024) |
| Hinge | 비공개 | 2024년 매출 $550M, YoY +38% |

Bumble은 2025년 유료 마케팅을 대폭 줄이고 **유기적 성장 전략으로 전환**을 선언했다.
이미 대형 앱들도 광고로 성장하는 방식에 한계를 느끼고 있다는 신호.

리퍼럴로 들어온 유저는 광고로 들어온 유저보다 **획득 비용 50% 저렴**, **LTV 16% 높다**.

### K-factor: 바이럴의 강도를 측정하는 수식

```
K = 사용자당 평균 초대 수 (i) × 초대→가입 전환율 (c)

K > 1.0   → 유료 광고 없이 자생적 성장 가능
K 0.3~0.7 → CAC 30~70% 절감 (여기만 해도 충분히 의미 있음)
```

| 사례 | K-factor | 결과 |
|---|---|---|
| Dropbox | ~0.7 | 15개월 만에 10만 → 400만 유저 (+3,900%) |
| Locket | 추정 >1.0 | 2주 200만 가입, 광고비 ≈ 0 |
| Uber 리퍼럴 | 비공개 | ROI 12배 |

**데이팅앱에서 K > 1이 어려운 이유와 내친소의 해결책**

매칭이 성공하면 두 명이 동시에 앱을 떠난다 → 전파자 풀이 줄어든다.
해결책: 매칭이 성사되어도 **주선자는 남아야 한다** → 역할 통합이 단순한 UX 문제가 아닌 이유.

### 보상 설계 원칙 (Dropbox·Uber에서 검증된 것)

1. **양면 보상** — 주선자와 소개받는 사람 모두 혜택
2. **앱 가치와 연결된 보상** — 현금보다 씬구점·프로필 열기 같은 핵심 기능
3. **이미 하던 행동을 공식화** — 카카오톡으로 하던 소개를 내친소 버튼 하나로

## 한국 시장이 특이한 이유

### 현황

| 앱 | 규모 | 특징 |
|---|---|---|
| 위피 | 누적 663만, 설치자 33만 | **2023년 한국 데이팅앱 매출 1위** |
| 틴더 | 설치자 42만 | 글로벌 인지도 |
| 아만다 | 누적 500만+ | 기존 유저 심사 통과해야 가입 |
| 글램 | 설치자 28만 | |

### 위피가 1위가 된 이유 — 내친소가 참고할 포인트

위피는 **데이팅앱이 아닌 소셜앱으로 포지셔닝**했다.
"동네에서 놀 사람 찾기", "지역 이벤트" 기능을 추가하면서 데이팅에 거부감 있는 유저도 유입 가능하게 만들었다. 이게 CAC를 낮추는 핵심 전략이었다.

### 소개팅 문화의 디지털화

한국은 오프라인 소개팅 문화가 강하다.
앱이 그 문화를 **대체**하려 하면 저항이 생기고, **디지털화**하면 자연스럽게 흡수된다.
내친소의 "친구 추천 기반 신뢰 시스템"은 이 문화와 가장 정확히 맞는 구조다.

### 6:4 성비가 무기인 이유

업계 평균 성비는 남성 60% / 여성 40%.
Bumble이 업계에서 이례적으로 성공한 건 여성 비율을 높게 유지했기 때문이다.

여성 리텐션이 올라가면 → 남성 매칭률이 올라가고 → 남성 리텐션도 올라간다.
내친소의 6:4 여성 우위 성비는 지켜야 할 핵심 자산이다.

## 우선순위 정리

| 순위 | 방향 | 왜 | 기대 효과 |
|---|---|---|---|
| 🔴 필수 | 주선 + 소개받기 역할 통합 | Ship이 실패한 이유, Wingman이 살아남은 이유 | 주선자 리텐션 확보 → LTV 증가 |
| 🟠 높음 | 소개 성사 시 양면 씬구점 보상 | Dropbox 3,900%, Uber 12x ROI | K-factor 0.3~0.7 → CAC 30~70% 절감 |
| 🟡 중간 | 카카오 공유로 프로필 미리보기 | Locket "친구가 있어야 작동" 구조 | 앱 밖 바이럴 → 신규 유입 루프 |
| 🟢 고려 | 공통 친구 있는 카드 상단 노출 | Cerca 소셜 그래프 매칭 방식 | 매칭 품질 향상 → 리텐션 개선 |
| 🔵 탐구 | "오늘 주선 가능" 시간 희소성 토글 | Thursday 목요일 한정 매칭 | 주선자 참여율 상승 |
