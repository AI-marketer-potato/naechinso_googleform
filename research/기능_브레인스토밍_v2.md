## MVP — Phase 1 (0~4주)

> 핵심 주선 루프가 돌아가는 최소 기능 세트

### 1. 피드 즉시 공유 ("이 사람 어때?")

- **뭘 하는 기능**: 피드 카드 보다가 → "친구에게 소개" 버튼 → 친구 선택 → 한 줄 메모(선택) → 전송. 3초 이내 완료
- **출처**: TikTok 영상 공유, Instagram DM 공유, Hinge "Send Profile"
- **왜 MVP**: 이게 없으면 주선 자체가 불가능. 가장 기본적인 주선 행동
- **핵심 UX**: 메시지 없이도 전송 가능 (TikTok 방식). 프리셋 메시지 3~4개 + 자유입력

```
피드 카드 → [친구에게 소개] 버튼
→ 친구 목록 (최근 대화 순)
→ 프리셋: "너 스타일이야 ㅋㅋ" / "취향 저격 아니야?" / "진지할 것 같아서"
→ 전송 → 썬구리 +5
```

---

### 4. 추천사 먼저 보기

- **뭘 하는 기능**: 소개받았을 때 프로필 사진이 아닌 **친구의 추천사가 먼저** 보이는 카드 구조
- **출처**: Cerca (공통 친구 → 배경 → 사진 순서), S'More (외모 블러 후 대화로 언블러)
- **왜 MVP**: 내친소의 "친구 보증" 철학을 시각적으로 보여주는 핵심 차별점
- **핵심 UX**: 카드 상단에 "민준이가 소개해줬어요" + 추천 한마디 → 스크롤하면 프로필

```
┌─────────────────────┐
│  👤 민준이가 소개    │
│  "걔 진짜 성격 좋아" │
│─────────────────────│
│  [프로필 보기 →]     │
│  볼게!  /  나중에    │
└─────────────────────┘
```

---

### 15. 주선 결과 피드백 루프

- **뭘 하는 기능**: 소개 후 진행 상황을 주선자에게 알림. "둘이 대화 시작했어!" / "매칭 성사!"
- **출처**: Tinder Matchmaker의 **실패 보완** — 추천자가 결과를 모르면 동기 소멸
- **왜 MVP**: Ship이 망한 이유가 "주선자가 계속 써야 할 이유 없음". 피드백이 주선 행동을 반복시킴
- **핵심 UX**: 프라이버시 보호 — 대화 내용은 안 보이고 상태만 전달

```
[주선자에게 알림]
"너의 소개로 태양 ↔ 지수가 대화를 시작했어! 🎉"
→ 썬구리 +20

[매칭 성사 시]
"주선 성공! 🎊 태양이 고마워하고 있어"
→ 썬구리 +50 + 뱃지
```

---

### 16. "소개받고 싶어요" 토글

- **뭘 하는 기능**: 프로필에 토글 — 켜면 친구가 나를 소개 가능, 끄면 불가
- **출처**: Hinge Matchmaker 2017 실패 교훈 (opt-out 기본값 → "creepy"), Thursday (시간 제한 활성화)
- **왜 MVP**: 다른 모든 주선 기능의 **안전장치**. 이게 없으면 Hinge처럼 "creepy" 반응 나옴
- **핵심 UX**: 기본값 OFF. 켤 때 "친구들이 너를 소개할 수 있게 돼!" 안내

```
설정 > 소개 관리
[🔘 소개받고 싶어요]  ← ON
"친구들이 다른 사람에게 너를 소개할 수 있어요.
 소개 전에 항상 네 동의를 먼저 구해요."
```

## Phase 2 (1~2개월)

> 코어 루프 강화 + 신뢰 시스템 + 외부 바이럴

### 6. 공통 친구 우선 노출

- **뭘 하는 기능**: 피드 카드에 "공통 친구 2명" 뱃지 표시. 공통 친구 있는 카드가 피드 상단에 올라옴
- **출처**: Cerca (프로필 순서: 공통 친구 → 배경 → 사진), LinkedIn (공통 연결 수 표시)
- **왜 Phase 2**: 공통 친구 데이터가 쌓여야 의미가 있음. MVP에서 주선이 일어나야 데이터 생김

```
┌────────────────────┐
│ 👥 공통 친구 2명    │  ← 카드 상단
│ 민준 · 태양         │
│────────────────────│
│ [프로필 사진]       │
│ 지수, 26            │
└────────────────────┘
```

---

### 12. 추천사 스택 (누적 보증)

- **뭘 하는 기능**: 받은 추천사가 프로필에 누적 표시. "친구 5명이 보증함" 뱃지. 추천사 많을수록 피드 우선 노출
- **출처**: LinkedIn endorsement (스킬별 보증 횟수 누적)
- **왜 Phase 2**: #4(추천사 먼저 보기)가 먼저 있어야 추천사의 가치가 생김

```
프로필 하단:
━━━ 친구 5명이 보증 ━━━
🗣 "걔 진짜 성격 좋아" — 민준
🗣 "유머 감각 최고" — 현우
🗣 "진지하게 연애하고 싶어하는 애" — 도윤
[+2 더보기]
```

---

### 3. 양방향 동시 소개

- **뭘 하는 기능**: 내 싱글 친구 A와 B를 골라서 동시에 소개. 양쪽 모두 수락하면 연결
- **출처**: Hinge Matchmaker (양방향 아이스브레이커), LinkedIn "Get Introduced"
- **왜 Phase 2**: #1(피드 즉시 공유)이 "카드→친구" 일방향이라면, 이건 "친구↔친구" 양방향. 더 복잡하지만 한국 소개팅 문화에 가장 가까움

```
[내 친구 중 싱글]
태양 👤     ↔     지수 👤
  ↑                  ↑
  └── 주선자: 나 ──┘

"이 두 사람 잘 어울릴 것 같아!"
→ 양쪽에게 동시 알림
→ 양쪽 수락 → 채팅 시작
```

---

### 5. 카카오톡 소개 카드

- **뭘 하는 기능**: 내친소 카드를 카카오톡으로 공유. 비유저가 미리보기 → 앱 설치 → 소개 카드 확인
- **출처**: Locket (앱이 작동하려면 상대도 설치 필요 → 자연 바이럴), Hinge Matchmaker (비유저 초대 링크)
- **왜 Phase 2**: 앱 내 기능이 먼저 완성돼야 외부 공유가 의미 있음

```
[카카오톡 공유 카드]
┌─────────────────────┐
│ 내친소 💌            │
│ "민준이가 너한테     │
│  소개해주고 싶은     │
│  사람이 있대!"       │
│                     │
│ [내친소에서 확인 →]  │
└─────────────────────┘
```

---

### 13. 구체적 프로필 반응

- **뭘 하는 기능**: 친구에게 소개할 때 프로필의 **특정 부분을 하이라이트**해서 보냄
- **출처**: Hinge (특정 사진/답변에 반응 → "이 사진 웃겼어요"), Letterboxd (짧은 리뷰 OK)
- **왜 Phase 2**: #1(피드 즉시 공유)의 업그레이드 버전

```
소개할 때:
"야 이 사람 3번째 여행 사진 봐, 너 취향이잖아"
→ 해당 사진이 하이라이트된 상태로 친구에게 전달
→ 받는 사람이 처음 보는 건 그 사진 + 친구 한마디
```

## Phase 3 (3~6개월)

> 스케일 기능 + 앱 밖 확장 + 커뮤니티

### 2. 친구 대신 피드 탐색

- **뭘 하는 기능**: 친구를 선택하면 → 그 친구의 선호 조건으로 필터링된 피드가 열림 → 대신 찾아주기
- **출처**: Ship (53% 매칭이 크루 대리 스와이프), Wingman (Wingman 모드)
- **왜 Phase 3**: "친구 조건 기반 필터링" 알고리즘 필요. 구현 복잡도 높음

---

### 7. 비유저 주선 참여

- **뭘 하는 기능**: 연애 중인 친구가 앱 설치 없이 **웹 링크로** 주선에 참여
- **출처**: Tinder Matchmaker (비유저 게스트 참여), Hinge Matchmaker (비유저 사용 가능)
- **왜 Phase 3**: 웹 경험 별도 구축 필요. 근데 임팩트는 큼 — "연애 중이지만 주선은 하고 싶은 사람"이 참여 가능

---

### 8. 소개팅 브릿지 채팅

- **뭘 하는 기능**: 매칭 성사 시 주선자가 3자 채팅방에 잠깐 들어와서 아이스브레이커 → 자연스러워지면 나가기
- **출처**: 한국 소개팅 문화 (카톡 단톡방 만들어주기), Hinge (자동 아이스브레이커)
- **왜 Phase 3**: 채팅 시스템 확장 필요. 3자 채팅방 → 2자 전환 UX 복잡

---

### 9. "얘 어때?" 투표

- **뭘 하는 기능**: 프로필 하나를 친구 여러 명에게 동시에 보내고 반응 모아보기
- **출처**: Ship crew, BeReal RealMoji
- **왜 Phase 3**: #1의 확장 버전. 여러 명에게 동시 전송 + 반응 집계 UI 필요

---

### 10. 하루 소개 제한 + 저녁 공개

- **뭘 하는 기능**: 주선자 하루 3건 제한. 소개받는 사람은 저녁 8시에 모아서 공개
- **출처**: Cerca (하루 4명, 오후 8시 공개), Thursday (목요일만), Coffee Meets Bagel (정오 큐레이션)
- **왜 Phase 3**: 유저 수가 충분해야 희소성이 작동. 적은 유저에서 제한하면 콘텐츠 부족

---

### 11. 주선 성공 공유 카드

- **뭘 하는 기능**: 매칭 성사 시 "나의 소개로 연결됐어요!" 공유 카드 자동 생성
- **출처**: Duolingo milestone 공유, Strava 운동 기록 공유
- **왜 Phase 3**: 성공 사례가 쌓여야 공유 카드가 의미 있음

---

### 14. 소개 = 입장권

- **뭘 하는 기능**: 신규 유저가 기존 유저의 소개로만 가입 가능. 월 3장 "소개 티켓"
- **출처**: Raya (수락률 8%), Clubhouse (초대장 2장)
- **왜 Phase 3**: 유저 풀이 충분해야 진입 장벽이 가치가 됨. 너무 빨리 걸면 성장 막힘 (Clubhouse 교훈)
- **주의**: Clubhouse처럼 초기 FOMO만 있고 핵심 가치 없으면 급락. 코어 기능이 단단해진 뒤 도입
