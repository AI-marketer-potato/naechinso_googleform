## 핵심 프레임워크 (기억할 것)

**설계 원칙 3개** (모든 기능 설계 시 적용)
1. 역할 통합 — 주선자/소개받는 모드 분리하지 말 것 (Ship 실패 이유)
2. 추천사 두 맥락 — 능동형 vs 세렌디피티형
3. 2×2 프레임 — 데이팅/비데이팅 × 내가소개/친구가소개

**기능 아이디어 A~J**
- A: 주선자 능동형 (A-1 친구 대신 피드, A-2 정성 추천사, A-3 양방향 소개)
- B: 주선자 세렌디피티형 (B-1 피드 즉시 공유 ★MVP, B-2 한 줄 메모, B-3 루틴)
- C: 소개받는 사람 (C-1 전용 카드, C-2 추천사 먼저 ★MVP, C-3 북마크)
- D: 보상/게이미피케이션 (D-1 뱃지, D-2 통계)
- E: 앱 밖 바이럴 (E-1 카카오 공유, E-2 소개 대기 초대)
- F: 한국 특화 (F-1 번개 모임, F-2 단톡방)
- G: 추천사 = 프로필 자산 (G-1 누적, G-2 요청)
- H: 네트워크 코드 (H-1 입장 코드, H-2 취향 코드)
- I: 유입 명분 (I-1 광고 카피, I-2 성공 스토리)
- J: 그룹 주선 (J-1 소개 그룹, J-2 취향 커뮤니티)

**썬구리 보상 타이밍 초안**
- 소개 전송: +5 🌟
- 상대 수락: +15 🌟
- 대화 시작(3메시지): +20 🌟
- 매칭 성사: +50 🌟 + 뱃지
- 소개받는 사람도: +10 🌟 (양면 보상)

**MVP 로드맵**
- Phase 1 (0~4주): B-1 피드 즉시 공유 + C-2 추천사 먼저 + D기본 보상
- Phase 2 (1~2개월): A-1 친구 대신 피드 + E-1 카카오 공유 + G-1 추천사 자산 + D-1 뱃지
- Phase 3 (3~6개월): H-1 네트워크 코드 + A-3 양방향 소개 + J-1 그룹 주선 + F-1 번개 모임
